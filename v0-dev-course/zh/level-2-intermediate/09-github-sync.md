# 连接 GitHub

## 什么是 GitHub？

GitHub 是存储和管理代码的平台。把它想象成代码的 Google Drive。它追踪每次更改，允许团队协作。

## 为什么要连接 v0 和 GitHub？

- **备份** — 代码安全存储
- **协作** — 与团队成员分享
- **版本控制** — 追踪所有更改历史
- **专业流程** — 行业标准做法

## 如何连接

1. 在 v0 项目中找到"Sync with GitHub"或 GitHub 图标
2. 登录 GitHub 账号
3. 创建新仓库或连接已有仓库
4. 授权 v0 向你的仓库推送代码

## 什么会被同步？

- 所有组件代码
- 样式和配置文件
- 项目结构
- 在 v0 中的更改会推送到 GitHub

## GitHub 基本概念

| 术语 | 含义 |
|------|------|
| Repository（仓库） | GitHub 上的项目文件夹 |
| Commit（提交） | 一次保存的更改 |
| Branch（分支） | 代码的平行版本 |
| Push（推送） | 把更改发送到 GitHub |
| Pull（拉取） | 从 GitHub 获取更改 |

## 技巧

- 给仓库起一个清晰的名字
- 提交时写简短说明
- 你不需要深入了解 Git——v0 处理复杂部分

## 总结

连接 GitHub 提供备份、协作和版本控制。v0 让这一切一键完成。下一课——部署到 Vercel。

# Подключение к GitHub

## Что такое GitHub?

GitHub — это платформа для хранения и управления кодом. Думайте о ней как о Google Диске, но для кода. Она отслеживает все изменения и позволяет работать в команде.

## Зачем подключать v0 к GitHub?

- **Резервная копия** — код в безопасности
- **Совместная работа** — делитесь с командой
- **Контроль версий** — вся история изменений
- **Профессиональный процесс** — стандарт индустрии

## Как подключить

1. В проекте v0 найдите «Sync with GitHub» или значок GitHub
2. Войдите в аккаунт GitHub
3. Создайте новый репозиторий или подключите существующий
4. Разрешите v0 отправлять код в ваш репозиторий

## Что синхронизируется?

- Весь код компонентов
- Стили и файлы конфигурации
- Структура проекта
- Изменения из v0 отправляются в GitHub

## Основные термины GitHub

| Термин | Значение |
|--------|----------|
| Repository (репозиторий) | Папка проекта на GitHub |
| Commit (коммит) | Сохранённое изменение |
| Branch (ветка) | Параллельная версия кода |
| Push (пуш) | Отправка изменений в GitHub |
| Pull (пулл) | Получение изменений из GitHub |

## Советы

- Давайте репозиторию понятное название
- Пишите короткие описания к коммитам
- Вам не нужно глубоко знать Git — v0 берёт сложную часть на себя

## Итог

Подключение к GitHub даёт резервное копирование, совместную работу и контроль версий. Дальше — публикация проекта.

# Работа с компонентами

## Что такое компоненты?

Компоненты — это переиспользуемые строительные блоки интерфейса. Кнопка — компонент. Панель навигации — компонент. Карточка товара — компонент.

Думайте о них как о кубиках LEGO — вы собираете страницы из компонентов.

## Зачем нужны компоненты

- **Переиспользование** — создал один раз, используешь везде
- **Единообразие** — одинаковый стиль кнопок на всех страницах
- **Скорость** — не нужно перерисовывать одно и то же

## Генерация отдельных компонентов

Вместо целых страниц можно просить v0 создать конкретные компоненты:

- "Create a notification badge component"
- "Build a user avatar with dropdown menu"
- "Make a search bar with autocomplete"
- "Design a file upload component with drag and drop"

## Типичные компоненты

| Компонент | Для чего |
|-----------|----------|
| Navbar | Навигация вверху страницы |
| Card | Блок с информацией |
| Modal | Всплывающее окно |
| Form | Поля ввода |
| Table | Таблица с данными |
| Sidebar | Боковая панель |
| Footer | Нижняя часть страницы |

## Сборка страницы из компонентов

Стройте страницу пошагово:

1. "Create a navbar with logo and menu links"
2. "Add a hero section below the navbar"
3. "Add a grid of product cards"
4. "Add a footer with social media links"

Каждый шаг дополняет предыдущий в одном чате.

## Итог

Компоненты — строительные блоки любого интерфейса. Генерируйте их по отдельности для большего контроля. Дальше — дизайн-системы.

# CASS 和 CM：构建代理记忆

## 记忆问题

AI 代理默认是无状态的 — 每次对话从零开始。就像一个每天早上都失忆的员工。ACFS 用 CASS 和 CM 解决这个问题。

### CASS — 对话代理会话搜索

CASS 记录和索引所有代理会话。

```bash
cass                          # 打开 TUI
cass search "数据库迁移"      # 搜索特定话题
```

### CASS 捕获什么

- 你发给代理的每个提示
- 代理的每个回复
- 创建或修改的文件
- 时间戳和会话元数据

### CM — CASS 记忆（程序性记忆）

CM 将原始会话历史提炼成 **可复用的知识**。

```bash
cm context "构建 REST API"  # 获取相关记忆
cm reflect                   # 更新程序性记忆
```

### 记忆如何随时间改善

**第1周**：你构建了 REST API。CM 记录模式、错误和解决方案。

**第3周**：你构建另一个 API。运行 `cm context "REST API"`，代理立即知道你的偏好和过去的错误。

**第2个月**：代理深入了解你的编码风格、项目结构偏好和常见陷阱。

### 实践技巧

1. 完成重要工作后运行 `cm reflect`
2. 在开始相关项目时使用 `cm context "主题"`
3. 忘记之前怎么解决的时候搜索 CASS
4. 记忆是积累的 — 用得越多越有价值

### 关键要点

CASS 记录所有代理会话。CM 将其提炼为可复用知识。重要工作后运行 `cm reflect`，相关项目前运行 `cm context`。

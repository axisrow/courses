# 飞轮循环：一切如何关联

## 核心概念

ACFS 中的「飞轮」不仅是名字 — 它是一个工作模式，每个循环都让下一个更好。

### 循环

```
计划 (Beads) → 协调 (Agent Mail) → 执行 (NTM + 代理)
    ↑                                          |
    |                                          ↓
    └──── 记忆 (CASS Memory) ← 扫描 (UBS) ←──┘
```

### 逐步说明

#### 1. 用 Beads 计划
用 Beads Viewer (`bv`) 把项目拆成任务。每个任务是一颗「珠子」，可以依赖其他珠子。

#### 2. 用 Agent Mail 协调
MCP Agent Mail 让代理互相沟通。代理 A 告诉代理 B：「数据库做好了——你可以开始 API 了。」

#### 3. 用 NTM + 代理执行
NTM 分配任务给代理。它们并行工作，各自在自己的 tmux 会话中。

#### 4. 用 UBS 扫描
代理写完代码后，UBS 在提交前扫描错误和安全问题。

#### 5. 用 CASS Memory 记忆
CASS 记录代理做的一切。下次遇到类似问题，代理可以查找之前有效的方法。

### 为什么是飞轮

- **第1轮**：代理从零开始写代码。慢。
- **第2轮**：代理记住了学到的东西。更快。
- **第5轮**：代理有了深度上下文。快得多。
- **第20轮**：代理几乎能预测你的需求。

### 关键要点

飞轮循环：计划 → 协调 → 执行 → 扫描 → 记忆 → 重复。每一轮改善下一轮，让你越来越快。

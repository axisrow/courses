# NTM：你的代理编排器

## 什么是 NTM？

**NTM**（Neuro-Tmux Manager）管理 AI 代理，就像指挥家管理乐团。不用手动为每个代理开终端输命令，NTM 从一个地方搞定一切。

### 为什么需要它

没有 NTM：
- 开终端 → 启动 Claude → 给任务 → 等待 → 切换终端 → 启动 Codex → 给任务……

有了 NTM：
- 打开 NTM → 同时给3个代理分配任务 → 它们并行工作

### 启动 NTM

```bash
ntm
```

打开 **TUI**（文本用户界面）— 在终端中运行的可视化程序。

### 核心概念

- **槽位（Slots）** — 代理运行的编号位置（像停车位）
- **提示（Prompts）** — 你发给代理的指令
- **广播（Broadcast）** — 同时给所有代理发送相同指令

### 基本工作流

1. 打开 NTM：`ntm`
2. 在槽位中启动代理
3. 向一个或所有代理发送提示
4. 观察它们并行工作
5. 查看结果

### 命令面板

按快捷键打开命令面板 — 所有可用操作的快速菜单。

### 何时使用 NTM

- 项目需要多个视角
- 需要比较不同 AI 代理的方案
- 想要并行化工作（前端 + 后端 + 测试）

### 关键要点

NTM 让你从一个界面管理多个 AI 代理。启动代理、广播提示、观察并行工作。这是你的任务控制中心。

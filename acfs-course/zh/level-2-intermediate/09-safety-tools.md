# 安全工具：DCG、SLB 和 UBS

## 为什么安全很重要

AI 代理很强大但可能很危险。一个被告知「清理代码」的代理可能会删除重要文件。ACFS 包含三个安全工具来防止灾难。

### DCG — 危险命令守卫

**功能**：在执行前拦截破坏性命令。

```bash
# 如果代理试图运行：
rm -rf /important-project

# DCG 在亚毫秒内拦截
# ⛔ 已拦截：破坏性文件系统操作
```

DCG 像夜店的保安 — 检查每个命令，拦截危险的。作为 Claude Code 的钩子工作。

### SLB — 安全保险箱

**功能**：对「核级」操作要求确认。

就像发射导弹的双钥匙系统 — 某些危险操作需要人类明确批准：
- 删除仓库
- 强制推送到 main 分支
- 删除数据库

### UBS — 通用缺陷扫描器

**功能**：扫描代码中的错误、安全漏洞和不良模式。

```bash
# 扫描当前项目
ubs .
```

### 三层保护协同工作

```
代理想要执行命令
        ↓
   DCG：是否破坏性？→ 拦截
        ↓（如果安全）
   SLB：是否核级？→ 询问人类
        ↓（如果批准）
   命令执行
        ↓
   UBS：扫描结果
```

### 关键要点

ACFS 用三层保护你：DCG 即时拦截危险命令，SLB 要求关键操作的人类批准，UBS 扫描代码缺陷。让代理自由工作，知道有这些护栏在。

# CASS и CM: память агентов

## Проблема памяти

AI-агенты по умолчанию не помнят ничего — каждый разговор начинается с нуля. Это как сотрудник, который каждое утро всё забывает. ACFS решает это через CASS и CM.

### CASS — поиск по сессиям агентов

CASS записывает и индексирует все сессии.

```bash
cass              # Открыть TUI
cass search "миграция базы данных"  # Поиск по теме
```

### Что записывает CASS

- Каждый промпт агенту
- Каждый ответ агента
- Созданные и изменённые файлы
- Временные метки и метаданные

### CM — процедурная память

CM превращает историю сессий в **переиспользуемые знания**.

```bash
cm context "Создание REST API"  # Получить релевантные воспоминания
cm reflect                       # Обновить память
```

### Как память улучшается

**Неделя 1**: Вы строите REST API. CM записывает паттерны и решения.

**Неделя 3**: Вы строите ещё один API. `cm context "REST API"` — и агенты сразу знают ваши предпочтения и прошлые ошибки.

**Месяц 2**: Агенты глубоко знают ваш стиль, структуру проектов и типичные подводные камни.

### Практические советы

1. Запускайте `cm reflect` после завершения значимой работы
2. Используйте `cm context "тема"` в начале похожих проектов
3. Ищите в CASS, когда не помните решение
4. Память накапливается — чем больше используете, тем ценнее

### Главный вывод

CASS записывает все сессии. CM превращает их в знания. Запускайте `cm reflect` после крупной работы и `cm context` перед похожими проектами.

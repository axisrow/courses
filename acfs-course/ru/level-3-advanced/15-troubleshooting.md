# Устранение неполадок и лучшие практики

## Частые проблемы и решения

### «Команда не найдена»

```bash
source ~/.zshrc    # Перезагрузить конфигурацию
exec zsh           # Или запустить новую оболочку
```

### «Агент не запускается»

Проверьте API-ключи:

```bash
echo $ANTHROPIC_API_KEY  # Claude
echo $OPENAI_API_KEY     # Codex
echo $GOOGLE_API_KEY     # Gemini
```

### «Установка прервалась»

Просто запустите снова — установщик идемпотентный:

```bash
curl -fsSL "https://raw.githubusercontent.com/Dicklesworthstone/agentic_coding_flywheel_setup/main/install.sh" | bash -s -- --yes --mode vibe
```

### «Tmux-сессия пропала»

```bash
tmux ls  # Если пусто — сервер перезагружался? Код на месте, создайте новые сессии.
```

### Диагностика

```bash
acfs doctor        # Полная проверка
which claude       # Путь к инструменту
claude --version   # Версия
```

## Лучшие практики

### Ежедневный рабочий процесс
1. SSH на VPS
2. Подключиться к tmux: `tmux attach`
3. Управлять агентами через NTM
4. `acfs doctor` раз в неделю

### Проектный рабочий процесс
1. Новая ветка для каждой фичи
2. Назначить агентов через NTM
3. Agent Mail для координации
4. UBS перед мержем
5. `cm reflect` после завершения

### Привычки безопасности
1. Делайте снапшоты VPS перед большими изменениями
2. Не используйте vibe mode на продакшне
3. Проверяйте код агентов перед деплоем
4. Не отключайте DCG и SLB
5. Коммитьте часто — это ваша кнопка «отмена»

### Когда начать заново

Иногда самое быстрое — чистый старт:
1. Создать новый VPS
2. Запустить ACFS
3. Клонировать репо
4. Через 30 минут вы снова в деле

Код живёт в Git, VPS одноразовый — в этом красота подхода ACFS.

### Главный вывод

Большинство проблем решается перезагрузкой оболочки или повторным запуском установщика. `acfs doctor` для диагностики. Код в Git, VPS одноразовый.

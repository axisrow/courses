# Git-стратегия и управление репозиториями

## Зачем Git в агентном кодировании

Когда несколько AI-агентов пишут код одновременно, нужен надёжный контроль версий.

### Базовая модель Git

Git — как система сохранений в видеоигре:
- **Commit** = сохранение игры
- **Branch** = создание альтернативной линии
- **Merge** = объединение двух линий
- **Push** = загрузка сохранений в облако

### Рабочий процесс Git в ACFS

```
main (рабочий код)
    ├── feature/agent-claude-backend
    ├── feature/agent-codex-frontend
    └── feature/agent-gemini-tests
```

Каждый агент работает в своей ветке. Это предотвращает конфликты.

### RU — утилита для репозиториев

```bash
ru sync    # Синхронизировать все репо
ru commit  # AI-коммиты по всем изменённым репо
```

При 10+ проектах RU экономит огромное количество времени.

### Лучшие практики

1. **Одна ветка на агента на задачу** — предотвращает конфликты
2. **Коммитьте часто** — маленькие частые сохранения лучше редких огромных
3. **Используйте резервирование файлов в Agent Mail**
4. **Сканируйте перед слиянием** — UBS на каждую ветку
5. **Проверяйте код агентов** — всегда ревьюйте перед мержем в main

### CAAM — менеджер аккаунтов

```bash
caam switch work      # Рабочий аккаунт
caam switch personal  # Личный аккаунт
```

### Главный вывод

Отдельные ветки для каждого агента. RU автоматизирует управление репо. CAAM переключает идентичности. Всегда проверяйте код агентов перед мержем.

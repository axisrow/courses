# Инструменты безопасности: DCG, SLB и UBS

## Почему безопасность важна

AI-агенты мощные, но могут быть опасны. Агент, которому сказали «почисти код», может удалить важные файлы. ACFS включает три инструмента безопасности.

### DCG — Защита от опасных команд

**Что делает**: Блокирует разрушительные команды до выполнения.

```bash
# Если агент попытается выполнить:
rm -rf /important-project

# DCG блокирует за доли миллисекунды
# ⛔ Заблокировано: разрушительная операция
```

DCG работает как охранник на входе — проверяет каждую команду и останавливает опасные. Работает как хук для Claude Code.

### SLB — Сейф безопасности

**Что делает**: Требует подтверждения для «ядерных» операций.

Как система двух ключей для запуска ракет — определённые опасные действия требуют одобрения человека:
- Удаление репозиториев
- Force-push в main-ветку
- Удаление баз данных

### UBS — Универсальный сканер ошибок

**Что делает**: Сканирует код на ошибки, уязвимости и плохие паттерны.

```bash
# Сканировать текущий проект
ubs .
```

### Три уровня защиты вместе

```
Агент хочет выполнить команду
        ↓
   DCG: Разрушительная? → Блокировать
        ↓ (если безопасна)
   SLB: Ядерная? → Спросить человека
        ↓ (если одобрена)
   Команда выполняется
        ↓
   UBS: Сканировать результат
```

### Главный вывод

ACFS защищает тремя уровнями: DCG блокирует опасные команды, SLB требует одобрения для критических операций, UBS сканирует код. Агенты работают свободно, а вы защищены.

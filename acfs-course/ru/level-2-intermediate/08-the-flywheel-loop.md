# Цикл маховика: как всё связано

## Основная концепция

«Маховик» в ACFS — это не просто название. Это рабочий паттерн, где каждый цикл работы улучшает следующий.

### Цикл

```
Планирование (Beads) → Координация (Agent Mail) → Выполнение (NTM + Агенты)
        ↑                                                      |
        |                                                      ↓
        └──── Запоминание (CASS Memory) ← Проверка (UBS) ←───┘
```

### Пошагово

#### 1. Планирование с Beads
Разбейте проект на задачи с помощью Beads Viewer (`bv`). Каждая задача — «бусина», которая может зависеть от других.

#### 2. Координация через Agent Mail
MCP Agent Mail позволяет агентам общаться. Агент А говорит Агенту Б: «Я закончил базу данных — можешь начинать API».

#### 3. Выполнение через NTM + Агенты
NTM назначает задачи агентам. Они работают параллельно, каждый в своей tmux-сессии.

#### 4. Проверка с UBS
После написания кода UBS проверяет его на ошибки и уязвимости перед коммитом.

#### 5. Запоминание с CASS Memory
CASS записывает всё, что делали агенты. В следующий раз агенты смогут посмотреть, что сработало раньше.

### Почему это маховик

- **Цикл 1**: Агенты пишут код с нуля. Медленно.
- **Цикл 2**: Агенты помнят, чему научились. Быстрее.
- **Цикл 5**: Агенты имеют глубокий контекст. Намного быстрее.
- **Цикл 20**: Агенты почти предугадывают ваши потребности.

### Главный вывод

Цикл маховика: Планирование → Координация → Выполнение → Проверка → Запоминание → Повтор. Каждый цикл улучшает следующий.

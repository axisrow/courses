# Аутентификация пользователей

## Что такое аутентификация?

Аутентификация позволяет пользователям создавать аккаунты и входить в приложение. Каждый пользователь видит свои данные и настройки.

## Добавление аутентификации

Bolt.new имеет встроенное управление пользователями. Просто попросите:

> «Добавь регистрацию и вход в приложение. Пользователи регистрируются по email и паролю. После входа они видят свою личную панель.»

## Что создаст Bolt

- **Страница регистрации** — для новых пользователей
- **Страница входа** — для существующих
- **Безопасность паролей** — автоматическое шифрование
- **Управление сессиями** — пользователь остаётся в системе
- **Защищённые страницы** — видны только авторизованным

## Вход через соцсети

> «Добавь возможность входа через Google и GitHub»

Пользователи смогут входить через существующие аккаунты без нового пароля.

## Роли пользователей

Для сложных приложений можно добавить роли:

> «Создай две роли: "администратор" и "обычный пользователь". Администраторы видят все данные и управляют пользователями. Обычные видят только свои данные.»

## Полезные функции

- Подтверждение email
- Сброс пароля
- Редактирование профиля
- Удаление аккаунта
- «Запомнить меня»

## Советы по безопасности

- Используйте встроенную аутентификацию Bolt — не пытайтесь делать свою
- Просите подтверждение email для защиты от фейковых аккаунтов
- Добавьте требования к сложности пароля

## Практическое задание

Добавьте аутентификацию в приложение контактов из предыдущего урока. Каждый пользователь должен видеть только свои контакты.

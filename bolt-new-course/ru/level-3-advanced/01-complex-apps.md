# Создание сложных многостраничных приложений

## Думаем масштабнее

Теперь, когда вы знаете основы, пора строить настоящие сложные приложения с множеством страниц, пользовательских сценариев и взаимосвязанных функций.

## Планирование приложения

Перед промптингом спланируйте структуру:

1. **Перечислите все страницы** — Главная, Дашборд, Настройки и т.д.
2. **Определите сценарии** — Что делает пользователь пошагово?
3. **Определите данные** — Какая информация хранится и отображается?
4. **Нарисуйте связи** — Как страницы связаны друг с другом?

## Строим поэтапно

### Фаза 1: Базовая структура
> «Создай приложение для управления проектами со страницами: Дашборд, Список проектов, Страница проекта, Список задач и Настройки. Настрой навигацию между всеми страницами.»

### Фаза 2: Добавляем данные
> «Добавь базу данных для проектов (название, описание, статус, дедлайн) и задач (заголовок, ответственный, приоритет, статус). У каждого проекта много задач.»

### Фаза 3: Добавляем функции
> «На Дашборде покажи: общее число проектов, задачи на эту неделю, график выполнения задач и последнюю активность.»

### Фаза 4: Полировка
> «Добавь анимации при переходе между страницами, скелетоны загрузки и состояния "пусто", когда данных ещё нет.»

## Управление контекстом

Для больших проектов помогайте ИИ:

- Указывайте конкретные страницы: «На странице Проекты...»
- Будьте точны в компонентах: «В боковой навигации...»
- Используйте одинаковые названия

## Пример реального приложения

Полная система управления фрилансера:
- Список клиентов с контактами
- Отслеживание проектов по клиентам
- Генерация счетов
- Учёт времени
- Дашборд с обзором заработка

## Практическое задание

Спланируйте и создайте приложение на 5 страниц. Перед промптингом выпишите все страницы, данные и функции.

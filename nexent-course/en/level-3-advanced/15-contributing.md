## Ways to Contribute

### Report Bugs

Found something broken? Open an issue on [GitHub](https://github.com/ModelEngine-Group/nexent/issues) with:

- Steps to reproduce the problem.
- Expected vs. actual behavior.
- Your environment (OS, Docker version, browser).

### Suggest Features

Have an idea? Check the [Feature Map](https://github.com/orgs/ModelEngine-Group/projects/6) first â€” your idea might already be planned. If not, open a feature request.

### Submit Code

1. Fork the repository.
2. Create a feature branch.
3. Make your changes.
4. Write tests.
5. Submit a pull request.

Follow the [Contribution Guide](https://modelengine-group.github.io/nexent/en/contributing) for detailed instructions.

### Improve Documentation

Documentation lives alongside the code. Fix typos, clarify instructions, or add new guides.

### Build MCP Plugins

Create new tools for the MCP ecosystem and share them with the community.

## Building from Source

For developers who want to run Nexent from source:

1. Clone the repository.
2. Follow the [Developer Guide](https://modelengine-group.github.io/nexent/en/getting-started/development-guide).
3. The backend is Python-based.
4. The frontend uses modern JavaScript/TypeScript.

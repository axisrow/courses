# 参与贡献 Nexent

## 为什么要贡献？

Nexent 是一个基于 MIT 许可证的开源项目。贡献代码可以帮助社区、改进平台，早期贡献者还能获得徽章和奖品等认可。

## 贡献方式

### 报告 Bug

发现问题了？在 [GitHub](https://github.com/ModelEngine-Group/nexent/issues) 上提交 Issue，包括：

- 复现步骤。
- 期望行为与实际行为。
- 您的环境（操作系统、Docker 版本、浏览器）。

### 建议功能

有新想法？先查看 [功能路线图](https://github.com/orgs/ModelEngine-Group/projects/6)，如果没有，请提交功能请求。

### 提交代码

1. Fork 代码仓库。
2. 创建功能分支。
3. 编写代码。
4. 编写测试。
5. 提交 Pull Request。

详细说明请参考 [贡献指南](https://modelengine-group.github.io/nexent/en/contributing)。

### 改进文档

修复错别字、完善说明或添加新指南。

### 构建 MCP 插件

为 MCP 生态系统创建新工具并分享给社区。

## 从源码构建

开发者可以从源码运行 Nexent：

1. 克隆代码仓库。
2. 按照 [开发者指南](https://modelengine-group.github.io/nexent/en/getting-started/development-guide) 操作。
3. 后端基于 Python。
4. 前端使用现代 JavaScript/TypeScript。

## 社区渠道

- **Discord** — [加入社区](https://discord.gg/tb5H3S3wyv) 交流和获得支持。
- **GitHub Issues** — 用于报告 Bug 和功能请求。
- **微信** — 在 [Nexent 网站](https://nexent.tech/en/contact) 找到二维码。

## Nexent 架构

Nexent 主要由以下部分组成：

- **前端** — React/Next.js Web 应用，运行在 3000 端口。
- **后端** — Python 服务，处理 AI 逻辑、数据处理和 API。
- **Elasticsearch** — 存储文档分块，支持语义搜索。
- **Redis** — 管理缓存和会话状态。
- **Docker** — 所有组件通过容器运行，方便部署。

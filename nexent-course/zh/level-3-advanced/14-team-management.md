# 团队管理与租户

## 多用户支持

Nexent 专为团队设计。多个用户可以共享同一个安装，每人拥有自己的账户、智能体和知识库。

## 核心概念

| 概念 | 说明 |
|------|------|
| **租户** | 一个独立的工作空间——类似 Nexent 中的独立公司 |
| **用户** | 租户中拥有账户的人 |
| **角色** | 一组权限（管理员、编辑者、查看者） |
| **邀请** | 向租户添加新用户的方式 |

## 设置用户

1. 进入 **设置 → 用户管理**。
2. 点击 **邀请用户**。
3. 输入对方的邮箱地址。
4. 分配角色。
5. 发送邀请。

## 角色与权限

- **管理员** — 完全访问权限：管理用户、智能体、知识库和设置。
- **编辑者** — 可以创建和编辑智能体与知识库，但不能管理用户。
- **查看者** — 可以使用智能体和搜索知识库，但不能创建或修改。

## 共享与个人资源

- **全局知识库** — 租户中所有用户都可访问。
- **个人知识库** — 仅创建者可见。
- **智能体** — 可以在团队中共享，也可以保持私有。

## 租户隔离

每个租户完全隔离：

- 数据不在租户间共享。
- 一个租户的智能体不能访问另一个租户的知识库。
- 模型提供商配置可以按租户独立设置。

## 最佳实践

- **明确角色。** 不要给所有人管理员权限。
- **组织知识库。** 使用命名规范，方便团队查找。
- **定期检查访问权限。** 移除不再需要访问的用户。

## 练习

1. 创建一个新用户账户。
2. 分配有限的角色。
3. 与该用户共享一个智能体和知识库。
4. 验证他们可以使用共享资源但无法访问受限资源。

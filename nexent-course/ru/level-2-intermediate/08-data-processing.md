# Обработка данных

## Обзор

Nexent включает мощный движок обработки данных, способный работать с 20+ форматами файлов. Он извлекает текст, таблицы и изображения из документов и подготавливает их для использования в базах знаний.

## Как работает обработка

### 1. Загрузка

Вы загружаете файлы в базу знаний или прямо в чат.

### 2. Извлечение

Nexent извлекает содержимое:

- **Текст** — из текстовых документов.
- **Таблицы** — с сохранением структуры.
- **Изображения** — через OCR для извлечения видимого текста.

### 3. Разбиение на фрагменты

Длинные документы делятся на небольшие осмысленные фрагменты для быстрого и точного поиска.

### 4. Индексация

Фрагменты сохраняются в Elasticsearch с векторными представлениями, что позволяет искать по смыслу, а не только по ключевым словам.

### 5. Реферирование

AI-модель генерирует сводку документа, помогая агентам понять содержание каждой базы знаний.

## Пакетная обработка

Для больших объёмов Nexent поддерживает пакетную обработку:

- Загружайте несколько файлов одновременно.
- Обработка идёт в фоновом режиме.
- Прогресс можно отслеживать в интерфейсе.

## Советы по качеству данных

- **Чистые файлы** дают лучшие результаты. Убирайте водяные знаки и лишнее форматирование.
- **Структурированные документы** (с заголовками и разделами) обрабатываются лучше.
- **Высокое разрешение изображений** улучшает результаты OCR.

## Практическое задание

1. Загрузите PDF с таблицами и изображениями в базу знаний.
2. Задайте агенту вопрос, требующий чтения таблицы из документа.
3. Проверьте, что ответ содержит правильные данные из таблицы.

# Установка и первый запуск

## Что понадобится

| Ресурс | Минимум |
|--------|---------|
| **CPU** | 2 ядра |
| **RAM** | 6 ГБ |
| **ПО** | Docker и Docker Compose |

> **Что такое Docker?** Docker — это инструмент для запуска приложений в изолированных контейнерах. Если у вас его нет, посетите [docker.com](https://www.docker.com/) и следуйте инструкциям для вашей ОС.

## Пошаговая установка

### 1. Скачайте Nexent

```bash
git clone https://github.com/ModelEngine-Group/nexent.git
```

### 2. Перейдите в папку Docker

```bash
cd nexent/docker
```

### 3. Создайте файл конфигурации

```bash
cp .env.example .env
```

Откройте `.env` в текстовом редакторе. Для базовой настройки достаточно заполнить данные голосового сервиса (или оставить пустыми, если голос не нужен).

### 4. Запустите Nexent

```bash
bash deploy.sh
```

Скрипт скачает и запустит все необходимые контейнеры. Первый запуск может занять несколько минут.

### 5. Откройте браузер

Перейдите на **http://localhost:3000**. Откроется мастер настройки.

## Мастер настройки

При первом запуске мастер проведёт вас через:

1. **Выбор языка** — английский или китайский.
2. **Подключение AI-модели** — понадобится API-ключ от провайдера моделей (OpenAI, SiliconFlow и др.).
3. **Базовые настройки** — имя рабочего пространства и значения по умолчанию.

## Остановка и перезапуск

Остановить Nexent:

```bash
cd nexent/docker
docker compose down
```

Запустить снова:

```bash
bash deploy.sh
```

Данные сохраняются между перезапусками.

## Решение проблем

| Проблема | Решение |
|----------|---------|
| Порт 3000 занят | Остановите другое приложение или измените порт в `docker-compose.yml` |
| Не хватает памяти | Закройте другие программы или увеличьте лимит памяти Docker |
| Контейнеры не запускаются | Выполните `docker compose logs` и проверьте ошибки |

Вы готовы к первому разговору!

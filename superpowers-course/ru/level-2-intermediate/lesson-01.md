# Subagent-Driven Development

## Введение

Как Superpowers использует субагентов для параллельной разработки.

## Как работают субагенты

Субагент — это отдельный экземпляр coding-агента, который получает:
- Контекст одной конкретной задачи
- Инструкции по TDD
- Критерии приёмки

### Двухэтапная проверка

После выполнения задачи субагентом:
1. **Проверка спецификации** — соответствует ли результат плану?
2. **Проверка качества кода** — чистый ли код, нет ли антипаттернов?

## Преимущества

- **Изоляция контекста** — каждый субагент работает с минимальным контекстом
- **Параллельность** — несколько субагентов могут работать одновременно
- **Надёжность** — ошибка одного субагента не ломает всё

## Навык `dispatching-parallel-agents`

Позволяет запускать несколько субагентов параллельно для независимых задач:

```
Задачи 3, 4 и 5 не зависят друг от друга → 
запустить 3 субагента параллельно
```

## Пример

```
План: 10 задач
Задачи 1-2: последовательно (зависимости)
Задачи 3-5: параллельно (независимы)
Задачи 6-10: последовательно (зависят от 3-5)
```

## Итоги

- Субагенты работают изолированно над конкретными задачами
- Двухэтапная проверка обеспечивает качество
- Параллельные субагенты ускоряют разработку
- Агент может работать автономно часами

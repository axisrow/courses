# Настройка под проект

## Введение

Адаптация Superpowers под конкретный проект и команду.

## AGENTS.md

Файл `AGENTS.md` в корне проекта — это ваш основной способ настройки. Агент читает его при старте.

```markdown
# AGENTS.md

## Технологический стек
- TypeScript, Node.js 20
- PostgreSQL, Prisma ORM
- Jest для тестов

## Соглашения
- Все файлы в src/
- Тесты рядом с кодом: src/foo.ts → src/foo.test.ts
- Использовать ESM

## Запрещено
- Не используй any
- Не пиши console.log для отладки
```

## Настройка навыков

Superpowers позволяет влиять на поведение навыков через контекст проекта:

- Добавьте примеры хорошего кода в проект
- Опишите архитектурные решения в документации
- Укажите специфичные тестовые фреймворки

## Интеграция с существующим кодом

При работе с существующим проектом:

1. Убедитесь, что тесты проходят (`npm test`)
2. Agент будет использовать git worktrees для изоляции
3. Существующий код станет контекстом для новых задач

## Итоги

- AGENTS.md — главный файл настройки
- Описывайте стек, соглашения и запреты
- Superpowers адаптируется к вашему проекту
- Существующие тесты и код становятся контекстом

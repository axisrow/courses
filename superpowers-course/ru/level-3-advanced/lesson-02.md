# Создание кастомных навыков

## Введение

Как создать собственный навык для Superpowers.

## Структура навыка

```
skills/my-skill/
└── SKILL.md
```

### Формат SKILL.md

```markdown
# Название навыка

## Когда использовать
Описание триггеров активации.

## Процесс
Пошаговые инструкции для агента.

## Примеры
Конкретные примеры применения.

## Антипаттерны
Чего избегать.
```

## Навык `writing-skills`

Superpowers содержит мета-навык для создания новых навыков. Используйте его:

```
Я хочу создать навык для автоматического создания документации
```

Агент:
1. Применит brainstorming к навыку
2. Создаст спецификацию
3. Напишет SKILL.md по шаблону
4. Протестирует навык

## Принципы хорошего навыка

- **Конкретность** — точные инструкции, не общие советы
- **Триггеры** — чёткие условия активации
- **Проверяемость** — как убедиться, что навык работает
- **Минимализм** — только необходимое

## Пример: навык для документации

```markdown
# auto-documentation

## Когда использовать
Активируется после завершения задачи, если изменены публичные API.

## Процесс
1. Найди все изменённые публичные функции/классы
2. Проверь наличие JSDoc/docstring
3. Если отсутствует — добавь
4. Обнови README если изменился API

## Антипаттерны
- Не документируй приватные методы
- Не пиши очевидные комментарии
```

## Итоги

- Навык = файл SKILL.md с инструкциями
- Используйте мета-навык writing-skills
- Навыки должны быть конкретными и проверяемыми
- Тестируйте навыки на реальных задачах

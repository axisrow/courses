# Как работает Superpowers

## Введение

Рабочий процесс: спецификация → план → субагенты.

## Этап 1: Brainstorming (Спецификация)

Когда вы даёте задачу, агент **не начинает сразу кодить**. Он:

1. Задаёт уточняющие вопросы
2. Исследует альтернативы
3. Формирует дизайн-документ
4. Показывает его по частям для проверки

### Пример диалога

```
Вы: Сделай CLI для управления задачами

Агент: Отличная идея! Давайте уточним:
- Где хранить задачи — файл, SQLite, облако?
- Какие операции: создать, удалить, пометить готовой?
- Нужны ли приоритеты и теги?
```

## Этап 2: Планирование

После утверждения спецификации агент создаёт **план реализации**:

- Задачи по 2-5 минут каждая
- Точные пути к файлам
- Полный код для каждой задачи
- Шаги проверки

### Пример плана

```
Задача 1: Инициализация проекта
- Создать package.json
- Настроить TypeScript
- Проверка: npm run build проходит

Задача 2: Модель задачи
- Создать src/models/task.ts
- Написать тест test/task.test.ts
- Проверка: тесты проходят
```

## Этап 3: Субагенты

Агент запускает **отдельного субагента** на каждую задачу:

1. Субагент получает контекст задачи
2. Выполняет работу с TDD
3. Главный агент проверяет результат (двухэтапный review)
4. При проблемах — исправление или повтор

## Пример полного цикла

```
Вы: Создай REST API для блога

[Brainstorming: 5 минут вопросов и ответов]
[Спецификация: документ на 1 страницу]
[План: 12 задач]
[Выполнение: субагенты работают 1-2 часа автономно]
[Результат: работающий API с тестами]
```

## Итоги

- Superpowers работает в 3 этапа: спецификация → план → выполнение
- Агент всегда уточняет задачу перед началом работы
- План разбивает работу на мелкие задачи
- Субагенты выполняют задачи параллельно с проверкой

# Superpowers 的工作原理

## 简介

工作流程：规格说明 → 计划 → 子代理。

## 阶段 1：头脑风暴（规格说明）

当你给出任务时，代理**不会立即开始编码**。它会：

1. 提出澄清问题
2. 探索替代方案
3. 创建设计文档
4. 分段展示供审查

## 阶段 2：计划

规格说明批准后，代理创建**实施计划**：

- 每个任务 2-5 分钟
- 精确的文件路径
- 每个任务的完整代码
- 验证步骤

## 阶段 3：子代理

代理为每个任务启动**独立的子代理**：

1. 子代理接收任务上下文
2. 使用 TDD 工作
3. 主代理审查（两阶段审查）
4. 有问题则修复或重试

## 总结

- Superpowers 分 3 个阶段工作：规格说明 → 计划 → 执行
- 代理总是在开始之前澄清任务
- 计划将工作分解为小任务
- 子代理执行任务并接受审查

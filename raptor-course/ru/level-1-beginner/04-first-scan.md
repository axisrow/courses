## Первое сканирование

### Что такое сканирование?

Сканирование проверяет ваш исходный код на известные шаблоны уязвимостей **без запуска программы**. Это называется «статический анализ».

### Как запустить сканирование

1. Откройте RAPTOR в Claude Code
2. Введите команду:

```
/scan
```

3. RAPTOR запросит путь к вашему репозиторию
4. Он запустит **Semgrep** и **CodeQL**
5. Результаты появятся в структурированном отчёте

### Понимание результатов

Каждая находка включает:

- **Серьёзность** — насколько опасна (Критическая, Высокая, Средняя, Низкая)
- **Расположение** — файл и номер строки
- **Описание** — что за уязвимость
- **Категория** — тип проблемы (SQL-инъекция, XSS, захардкоженные секреты и т.д.)

### Пример: сканирование тестовых файлов

RAPTOR включает тестовые файлы для практики:

```
/scan --repo test/data/
```

Это найдёт намеренные уязвимости вроде SQL-инъекции и XSS.

### Что происходит за кулисами

RAPTOR использует **правила Semgrep** из `engine/semgrep/rules/`, покрывающие:

- SQL-инъекции
- Инъекции команд
- Обход путей (Path Traversal)
- Захардкоженные секреты
- Слабую криптографию
- SSRF

### Главный вывод

Первое сканирование — это просто ввод `/scan`. Начните с тестовых файлов, а затем проверьте собственный код.

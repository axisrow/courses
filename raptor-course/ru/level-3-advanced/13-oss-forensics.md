## Расследование OSS Forensics

### Что такое OSS Forensics?

Модуль OSS Forensics в RAPTOR расследует **публичные GitHub-репозитории** на предмет атак на цепочку поставок, бэкдоров и подозрительной активности.

### Команда `/oss-forensics`

```
/oss-forensics
```

### Источники доказательств

RAPTOR собирает доказательства из нескольких источников:

1. **GH Archive** — неизменяемые исторические данные всех публичных событий GitHub, запрашиваемые через BigQuery
2. **GitHub API** — текущее состояние репозитория
3. **Wayback Machine** — исторические снимки веб-страниц
4. **Локальный Git** — прямой анализ репозитория

### Процесс расследования

Модуль использует **мультиагентную архитектуру**:

1. **Сбор доказательств** — специализированные следователи работают параллельно
2. **Извлечение IOC** — автоматическое извлечение индикаторов компрометации
3. **Верификация доказательств** — проверка по оригинальным неизменяемым источникам
4. **Формирование гипотез** — AI-генерация гипотез с итеративным уточнением
5. **Криминалистический отчёт** — таймлайн, атрибуция и IOC

### Восстановление удалённого контента

RAPTOR может восстановить **удалённый контент**:
- Удалённые коммиты (доступны по SHA)
- Удалённые issues и комментарии (через GH Archive)
- Изменённые описания репозиториев (через Wayback Machine)

export GOOGLE_APPLICATION_CREDENTIALS=/path/to/service-account.json

### Реальный пример

Бэкдор XZ Utils (CVE-2024-3094) — идеальный пример, где инструменты вроде RAPTOR могут помочь расследовать хронологию атаки.

### Главный вывод

OSS Forensics — следственное подразделение RAPTOR. Оно превращает GitHub-репозитории в места преступлений и предоставляет доказательный анализ атак на цепочку поставок.

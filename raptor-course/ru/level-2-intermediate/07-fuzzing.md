## Фаззинг с RAPTOR

### Что такое фаззинг?

Фаззинг — это подача случайных или полуслучайных данных программе для поиска сбоев. Сбои часто указывают на уязвимости — переполнение буфера или повреждение памяти.

### Как RAPTOR фаззит

RAPTOR использует **AFL++** (American Fuzzy Lop) — один из самых мощных фаззеров:

1. Вы предоставляете скомпилированный бинарный файл
2. AFL++ генерирует тысячи мутированных входных данных
3. Он отслеживает, какие входы вызывают новые пути выполнения
4. Он сохраняет входы, вызывающие сбои

### Запуск фаззинга

```
/fuzz
```

RAPTOR запросит:
- **Путь к бинарному файлу**
- **Длительность** (по умолчанию: 1 час)
- **Начальные входные данные**

### Понимание результатов

После фаззинга RAPTOR сообщает:

- **Уникальные сбои** — различные способы вызвать сбой программы
- **Покрытие кода** — какой процент программы был протестирован
- **Анализ сбоев** — AI-объяснение причин каждого сбоя

### Команда `/crash-analysis`

Для глубокого исследования сбоев:

```
/crash-analysis
```

### Советы для эффективного фаззинга

1. **Предоставьте хорошие начальные данные** — примеры корректных входных данных
2. **Фаззьте дольше** — больше времени = больше покрытие = больше найденных ошибок
3. **Компилируйте с санитайзерами** — используйте AddressSanitizer для обнаружения ошибок памяти

### Главный вывод

Фаззинг находит ошибки, которые статический анализ не может обнаружить. Интеграция RAPTOR с AFL++ делает запуск фаззинга простым и удобным.

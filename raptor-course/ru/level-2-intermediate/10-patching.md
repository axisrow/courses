## Генерация патчей безопасности

### От уязвимости к исправлению

Найти и доказать ошибку — только половина работы. RAPTOR также может **генерировать патчи**.

### Команда `/patch`

```
/patch
```

### Что включает патч

- **Изменённый код** — конкретные необходимые изменения
- **Объяснение** — почему это исправление работает
- **Побочные эффекты** — возможное влияние на функциональность
- **Предложения по тестированию** — как проверить исправление

### Полный автономный процесс

Команда `/agentic` запускает всё последовательно:

```
/agentic
```

1. Сканирование → поиск уязвимостей
2. Анализ → оценка серьёзности с AI
3. Эксплойт → доказательство эксплуатируемости
4. Патч → генерация исправлений

### Python CLI для автоматизации

Для CI/CD пайплайнов используйте Python:

```bash
python3 raptor.py agentic --repo /path/to/code
```

### Лучшие практики

1. **Проверяйте патчи вручную** — AI-сгенерированные исправления всегда должны проверяться человеком
2. **Запускайте тесты после патча** — убедитесь, что исправление не сломало функциональность
3. **Документируйте изменение**
4. **Пересканируйте после патча** — убедитесь, что уязвимость действительно устранена

### Главный вывод

RAPTOR замыкает цикл от обнаружения до исправления. Команда `/agentic` даёт полный автономный процесс, но проверка патчей человеком остаётся обязательной.

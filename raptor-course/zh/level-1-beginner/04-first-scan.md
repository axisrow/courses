## 运行你的第一次扫描

### 什么是扫描？

扫描在**不运行程序**的情况下检查源代码中的已知漏洞模式。这称为"静态分析"。

### 如何运行扫描

1. 在 Claude Code 中打开 RAPTOR
2. 输入命令:

```
/scan
```

3. RAPTOR 会询问你的代码仓库路径
4. 它对你的代码运行 **Semgrep** 和 **CodeQL**
5. 结果以结构化报告呈现

### 理解结果

每个发现包括:

- **严重性** — 有多危险？（严重、高、中、低）
- **位置** — 哪个文件和行号
- **描述** — 漏洞是什么
- **类别** — 问题类型（SQL 注入、XSS、硬编码密钥等）

### 示例:扫描测试文件

RAPTOR 包含可供练习的测试文件:

```
/scan --repo test/data/
```

这将发现故意放置的漏洞，如 SQL 注入和 XSS。

### 幕后发生了什么

RAPTOR 使用存储在 `engine/semgrep/rules/` 中的 **Semgrep 规则**，涵盖:

- SQL 注入
- 命令注入
- 路径遍历
- 硬编码密钥
- 弱加密
- SSRF

### 核心要点

第一次扫描只需输入 `/scan`。从测试文件开始，然后扫描你自己的代码。

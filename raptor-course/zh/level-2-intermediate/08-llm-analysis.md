## AI 驱动的漏洞分析

### 为什么使用 AI？

传统工具发现潜在问题，但不能总是告诉你:
- 这个缺陷**真的**可以利用吗？
- 影响有多**严重**？
- 最好的**修复**方式是什么？

RAPTOR 使用大型语言模型（LLM）来回答这些问题。

### `/analyze` 命令

```
/analyze
```

运行 AI 分析，**不**生成漏洞利用代码或补丁。比完整工作流程快 50%，成本更低。

### AI 检查什么

1. **代码上下文** — 阅读周围代码以理解漏洞
2. **数据流** — 追踪恶意输入如何到达危险函数
3. **可利用性** — 评估攻击的实际可行性
4. **影响** — 估计被利用时的损害
5. **建议** — 提出具体修复方案

### LLM 提供商配置

RAPTOR 通过 **LiteLLM** 支持多个 AI 提供商:

- **Anthropic Claude** — 最佳整体质量
- **OpenAI GPT-4** — 强大的替代选择
- **Google Gemini** — 良好质量
- **Ollama** — 免费但漏洞利用生成质量较低

### 成本管理

RAPTOR 实时跟踪成本。设置预算以防止意外账单:

```python
config = LLMConfig(max_cost_per_scan=1.0)  # 每次扫描 $1 限制
```

典型成本:每个漏洞分析约 $0.03。

### 核心要点

AI 分析将原始扫描结果转化为可操作的情报。它不仅告诉你缺陷*是什么*，还告诉你*有多危险*以及*如何修复*。

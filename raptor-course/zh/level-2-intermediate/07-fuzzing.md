## 使用 RAPTOR 进行模糊测试

### 什么是模糊测试？

模糊测试是向程序输入随机或半随机数据以查看其是否崩溃的做法。崩溃通常表示安全漏洞，如缓冲区溢出或内存损坏。

### RAPTOR 如何进行模糊测试

RAPTOR 使用 **AFL++**（American Fuzzy Lop），最强大的模糊测试器之一:

1. 你提供一个编译好的二进制文件
2. AFL++ 生成数千个变异输入
3. 它监控哪些输入触发了新的代码路径
4. 它保存导致崩溃的输入

### 运行模糊测试

```
/fuzz
```

RAPTOR 会询问:
- **二进制文件路径**
- **持续时间**（默认:1小时）
- **种子输入** — 开始变异的示例输入

### 理解结果

模糊测试后，RAPTOR 报告:

- **唯一崩溃** — 使程序崩溃的不同方式
- **代码覆盖率** — 测试了程序的百分之多少
- **崩溃分析** — AI 对每次崩溃原因的解释

### `/crash-analysis` 命令

用于深入调查崩溃:

```
/crash-analysis
```

### 有效模糊测试的技巧

1. **提供好的种子输入** — 给出有效输入的示例
2. **测试更长时间** — 更多时间 = 更多覆盖 = 更多发现的缺陷
3. **使用 sanitizer 编译** — 使用 AddressSanitizer 捕获内存错误

### 核心要点

模糊测试能发现静态分析无法发现的缺陷。RAPTOR 与 AFL++ 的集成使设置和运行模糊测试变得简单。

## 生成安全补丁

### 从漏洞到修复

发现和证明缺陷只是工作的一半。RAPTOR 还可以**生成补丁**来修复发现的漏洞。

### `/patch` 命令

```
/patch
```

### 补丁包含什么

- **修改后的代码** — 所需的具体更改
- **解释** — 为什么这个修复有效
- **副作用** — 对功能的潜在影响
- **测试建议** — 如何验证修复有效

### 完整自主工作流程

`/agentic` 命令按顺序运行所有步骤:

```
/agentic
```

1. 扫描 → 发现漏洞
2. 分析 → 用 AI 评估严重性
3. 利用 → 证明可利用性
4. 修补 → 生成修复

### 用于自动化的 Python CLI

用于 CI/CD 流水线:

```bash
python3 raptor.py agentic --repo /path/to/code
```

### 最佳实践

1. **手动审查补丁** — AI 生成的修复应始终由人工审查
2. **打补丁后运行测试** — 确保修复不会破坏功能
3. **记录更改**
4. **打补丁后重新扫描** — 验证漏洞确实已修复

### 核心要点

RAPTOR 闭合了从发现到修复的循环。`/agentic` 命令提供完整的自主工作流程，但补丁的人工审查仍然至关重要。

# Добавление контекста к коллекциям

## Зачем нужен контекст?

Контекст — одна из самых мощных возможностей QMD. Когда вы добавляете описание к коллекции или папке, оно возвращается вместе с результатами поиска. Это помогает ИИ понять, *что за контент* он видит.

Без контекста ИИ видит: `notes/meeting.md — Score: 85%`

С контекстом: `notes/meeting.md — Context: "Заметки с еженедельных стендапов" — Score: 85%`

## Добавление контекста к коллекции

```sh
qmd context add qmd://notes "Личные заметки и идеи"
qmd context add qmd://meetings "Протоколы и записи встреч"
qmd context add qmd://docs "Рабочая документация"
```

## Контекст для подпапок

Контекст работает как дерево:

```sh
qmd context add qmd://docs/api "Документация API"
qmd context add qmd://notes/work "Рабочие заметки"
```

## Контекст из текущей папки

```sh
cd ~/notes
qmd context add "Личные заметки и идеи"
```

## Глобальный контекст

```sh
qmd context add / "База знаний по моим проектам"
```

## Просмотр и удаление

```sh
qmd context list
qmd context rm qmd://notes/old
```

## Лучшие практики

1. **Будьте описательны** — «Протоколы еженедельных стендапов команды за 2024» лучше, чем «встречи»
2. **Используйте иерархию** — общий контекст на коллекциях, детальный на подпапках
3. **Думайте об ИИ** — что нужно знать ИИ, чтобы понять эти документы?
4. **Будьте краткими** — одно чёткое предложение обычно достаточно

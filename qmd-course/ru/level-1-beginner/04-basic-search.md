# Основы поиска

## Три способа поиска

QMD даёт три команды поиска — от простой до самой мощной:

| Команда | Метод | Скорость | Когда использовать |
|---------|-------|----------|-------------------|
| `qmd search` | Ключевые слова (BM25) | Быстро | Вы знаете точные слова |
| `qmd vsearch` | Семантический (ИИ) | Средне | Поиск по смыслу |
| `qmd query` | Гибридный (всё вместе) | Медленнее | Лучшее качество |

## Поиск по ключевым словам

Самый быстрый вариант — находит документы с точными словами:

```sh
qmd search "план проекта"
```

## Семантический поиск

Понимает *смысл*, а не только слова. Может найти релевантные документы, даже если в них другие термины:

```sh
qmd vsearch "как развернуть приложение"
```

Например, поиск «как развернуть» может найти документ о «процессе релиза».

## Гибридный поиск (рекомендуется)

Сочетает поиск по словам, семантический поиск и ИИ-ранжирование:

```sh
qmd query "процесс квартального планирования"
```

## Понимание результатов

```
docs/guide.md:42 #a1b2c3
Title: Качество кода
Score: 93%

Этот раздел описывает подход к разработке
качественного программного обеспечения.
```

- **Путь** — расположение файла
- **Docid** — короткий идентификатор (`#a1b2c3`)
- **Score** — релевантность (чем выше, тем лучше)
- **Сниппет** — предпросмотр найденного контента

## Шкала оценок

| Оценка | Значение |
|--------|----------|
| 80–100% | Очень релевантно |
| 50–80% | Умеренно релевантно |
| 20–50% | Частично релевантно |
| 0–20% | Низкая релевантность |

## Поиск в конкретной коллекции

```sh
qmd search "API" -c notes
```

## Практика

1. Выполните `qmd search` со словом, которое есть в ваших документах
2. Попробуйте `qmd vsearch` с вопросом о содержимом
3. Сравните результаты `qmd search` и `qmd query`

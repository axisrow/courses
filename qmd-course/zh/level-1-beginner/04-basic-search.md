# 基本搜索

## 三种搜索方式

| 命令 | 方法 | 速度 | 适用场景 |
|------|------|------|----------|
| `qmd search` | 关键词（BM25） | 快 | 知道确切词语 |
| `qmd vsearch` | 语义（AI） | 中等 | 按概念和含义搜索 |
| `qmd query` | 混合（全部结合） | 较慢 | 最佳综合结果 |

## 关键词搜索

最快的选项，查找包含确切词语的文档：

```sh
qmd search "项目时间线"
```

## 语义搜索

理解*含义*，即使用词不同也能找到相关文档：

```sh
qmd vsearch "如何部署应用程序"
```

例如，搜索"如何部署"可能会找到关于"发布流程"的文档。

## 混合搜索（推荐）

结合关键词搜索、语义搜索和 AI 重排序：

```sh
qmd query "季度规划流程"
```

## 理解搜索结果

```
docs/guide.md:42 #a1b2c3
Title: 软件工艺
Score: 93%

本节介绍构建高质量软件的工艺方法。
```

- **路径** — 文件位置
- **Docid** — 短标识符（`#a1b2c3`）
- **Score** — 相关性（越高越好）
- **片段** — 匹配内容的预览

## 评分指南

| 评分 | 含义 |
|------|------|
| 80–100% | 高度相关 |
| 50–80% | 中度相关 |
| 20–50% | 部分相关 |
| 0–20% | 低相关性 |

## 限定集合搜索

```sh
qmd search "API" -c notes
```

## 练习

1. 用你文档中的词运行 `qmd search`
2. 用问题尝试 `qmd vsearch`
3. 比较 `qmd search` 和 `qmd query` 的结果

# 实际工作流程

## 为什么需要工作流程？

学会了QMD的各项功能后，下一步是把它们组合起来，形成适合你日常工作的流程。本课介绍几个常见场景。

## 场景一：团队知识库

**目标**：让团队成员随时查找公司内部文档。

步骤：
1. 创建一个集合，命名为 `team-docs`
2. 把公司手册、流程文档、常见问题解答导入进去
3. 设置定期同步，确保新文档自动加入
4. 团队成员通过自然语言提问来查找信息

**效果**：新员工不再需要反复问同事，直接问QMD就行。

## 场景二：个人研究助手

**目标**：管理和检索大量研究资料。

步骤：
1. 为每个研究主题创建独立的集合
2. 导入论文、笔记、网页摘录
3. 用语义搜索找到跨文档的关联
4. 用AI代理帮你总结和对比不同资料

**效果**：不再淹没在成堆的文件里，关键信息一问就到。

## 场景三：客服知识库

**目标**：帮助客服团队快速回答客户问题。

步骤：
1. 导入产品文档、历史工单、常见问题
2. 配置输出格式，让回答简洁易懂
3. 通过MCP/HTTP接口连接到你的客服系统
4. 客服人员输入客户问题，系统自动推荐答案

**效果**：回复速度更快，答案更一致。

## 场景四：定期报告生成

**目标**：定期从文档库中生成摘要报告。

步骤：
1. 保持集合中的数据持续更新
2. 编写查询模板，提取关键指标和变化
3. 用AI代理自动生成报告草稿
4. 人工审核后发布

**效果**：省去手动整理数据的时间。

## 设计自己的工作流程

问自己这些问题：
- 我最常查找什么类型的信息？
- 哪些步骤是重复的、可以自动化的？
- 谁是最终用户？他们的技术水平如何？

## 小结

QMD的价值不在于单个功能，而在于把这些功能串联成适合你的工作流程。从一个简单的场景开始，逐步扩展，你会发现它越来越好用。

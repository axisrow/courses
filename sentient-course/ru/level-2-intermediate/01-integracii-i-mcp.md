# Интеграции и MCP-серверы

## Что такое MCP?

MCP (Model Context Protocol) — способ подключения Sentient к внешним сервисам. Каждая интеграция реализована как небольшой MCP-сервер, который умеет:

1. Авторизоваться во внешнем сервисе (обычно через OAuth).
2. Предоставлять набор **инструментов**, которые может вызывать AI.
3. Возвращать результаты основному серверу Sentient.

## Доступные интеграции

| Категория | Сервисы |
|---|---|
| **Google Workspace** | Gmail, Calendar, Drive, Docs, Sheets, Slides, Tasks, Contacts, Maps |
| **Коммуникация** | Slack, Discord, WhatsApp |
| **Продуктивность** | Notion, Trello, GitHub |
| **Информация** | Google Search, News, AccuWeather |
| **Утилиты** | Файловый менеджер, QuickChart, Memory, History |

## Подключение интеграции

1. Откройте страницу **Integrations** в интерфейсе Sentient.
2. Нажмите **Connect** рядом с нужным сервисом.
3. Пройдите авторизацию OAuth.
4. После подключения Sentient сможет использовать сервис в разговорах и задачах.

## Как AI использует интеграции

Когда вы говорите «Проверь мой календарь на завтра», AI:
1. Определяет, что нужен инструмент Google Calendar.
2. Вызывает MCP-сервер `gcal`.
3. MCP-сервер использует ваш OAuth-токен для получения событий.
4. Результаты возвращаются AI, который форматирует их для вас.

## Оркестратор

Модуль `orchestrator` координирует сложные рабочие процессы с несколькими инструментами — например, прочитать почту, создать событие в календаре и отправить сводку в Slack.

## Итог

MCP-серверы — мост между Sentient и внешним миром. Они превращают его из чат-бота в **агента, способного действовать**.

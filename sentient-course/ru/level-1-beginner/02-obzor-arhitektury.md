# Обзор архитектуры

## Общая картина

Sentient состоит из двух основных частей:

1. **Клиент (Frontend)** — веб-приложение на Next.js. Это то, что вы видите в браузере.
2. **Сервер (Backend)** — приложение на Python (FastAPI). Это «мозг», который обрабатывает сообщения, общается с AI-моделями и подключается к внешним сервисам.

## Вспомогательные сервисы

| Сервис | Роль |
|---|---|
| **Redis** | Быстрый кеш и брокер сообщений для фоновых задач. |
| **PostgreSQL + pgvector** | Хранит данные и векторные представления для поиска по памяти. |
| **ChromaDB** | Дополнительная векторная база для семантического поиска. |
| **LiteLLM** | Прокси для работы с разными AI-моделями (OpenAI, Gemini, Ollama и др.). |
| **Celery Workers** | Выполняют фоновые и запланированные задачи. |

## Путь сообщения

1. Вы вводите сообщение в веб-интерфейсе.
2. Клиент отправляет его серверу через WebSocket.
3. Сервер проверяет вашу память и контекст, затем отправляет всё AI-модели через LiteLLM.
4. Модель отвечает. Если ей нужен инструмент (например, проверить календарь), сервер вызывает соответствующий **MCP-сервер**.
5. Финальный ответ возвращается в браузер.

## MCP Hub — уровень интеграций

MCP (Model Context Protocol) — это стандарт подключения к внешним приложениям. Каждый MCP-сервер знает, как авторизоваться и работать с одним конкретным сервисом.

## Итог

Sentient = **Веб-интерфейс** + **Python-сервер** + **базы данных** + **AI-модели** + **MCP-интеграции**. Понимание этих блоков поможет вам в дальнейших уроках.

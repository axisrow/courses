# 任务与自动化

## 超越聊天

Sentient 不仅限于实时对话。它可以运行**后台任务**——按计划执行、响应触发器或作为长时间运行的流程。

## 任务类型

| 类型 | 说明 | 示例 |
|---|---|---|
| **定时任务** | 在指定时间执行一次。 | "下午6点给我发今天邮件的摘要。" |
| **定期任务** | 按固定周期执行。 | "每周一从日历创建周计划。" |
| **触发任务** | 特定事件发生时执行。 | "收到老板的邮件时，通知我的 Slack。" |
| **群体任务** | 复杂任务分解为多个 AI 代理执行的子任务。 | "调研竞争对手并生成报告。" |

## 创建任务

1. 打开 **Tasks** 页面。
2. 点击 **New Task**。
3. 在 **Task Composer** 中描述任务。
4. 选择类型并配置计划或触发条件。
5. 保存。**Celery workers** 将负责执行。

## 查看任务历史

Tasks 页面显示：
- 活跃任务及下次运行时间。
- 已完成任务的结果。
- 失败任务的错误详情。

## 内部工作原理

1. 任务定义保存到数据库。
2. **Celery Beat**（调度器）检查到期任务。
3. **Celery Worker** 从 Redis 队列中取出任务执行。
4. Worker 使用与聊天相同的 AI + MCP 管道。

## 小结

任务将 Sentient 从被动的聊天机器人转变为主动的自动化引擎，全天候为你工作。

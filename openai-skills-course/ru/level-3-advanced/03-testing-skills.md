# Тестирование навыков

## Введение

Тестирование — важный этап создания навыка. Хорошо протестированный навык работает надёжно и предсказуемо. В этом уроке мы рассмотрим все аспекты тестирования.

## Шаг 1: Валидация структуры

Запустите `quick_validate.py`:

```bash
python scripts/quick_validate.py ./my-skill
```

Скрипт проверит:
- Наличие SKILL.md
- Корректность YAML frontmatter
- Наличие полей `name` и `description`
- Правильность именования (kebab-case)

## Шаг 2: Тестирование скриптов

Каждый скрипт в папке `scripts/` нужно запустить и проверить:

```bash
# Проверяем, что скрипт запускается
python scripts/my_script.py --help

# Тестируем на примере
python scripts/my_script.py test_data.csv

# Проверяем крайние случаи
python scripts/my_script.py empty.csv
python scripts/my_script.py large_file.csv
```

### На что обращать внимание:

- Скрипт не падает с ошибкой
- Вывод соответствует ожиданиям
- Обработаны пустые файлы и некорректные данные
- Работает с разными операционными системами (если нужно)

## Шаг 3: Тестирование в Codex

Установите навык локально и проверьте в реальных условиях:

```bash
# Установите навык
cp -r ./my-skill ~/.codex/skills/

# Перезапустите Codex
# Попробуйте разные запросы
```

### Что проверить:

1. **Триггер** — активируется ли навык на правильные запросы?
2. **Точность** — правильно ли выполняет задачу?
3. **Ложные срабатывания** — не активируется ли на посторонние запросы?

## Шаг 4: Оценки (Evaluations)

Некоторые курированные навыки включают папку `evaluations/` с JSON-файлами для автоматического тестирования.

Пример структуры файла оценки:

```json
{
  "prompt": "Подготовь заметки для статус-митинга",
  "expected_skill": "notion-meeting-intelligence",
  "expected_output_contains": ["шаблон", "повестка"]
}
```

## Шаг 5: Итеративное улучшение

После первого тестирования:

1. Используйте навык на реальных задачах
2. Замечайте проблемы и неточности
3. Обновляйте SKILL.md и скрипты
4. Тестируйте снова

## Чек-лист тестирования

- [ ] `quick_validate.py` проходит без ошибок
- [ ] Все скрипты запускаются и работают корректно
- [ ] Навык активируется на правильные запросы
- [ ] Нет ложных срабатываний
- [ ] Обработаны крайние случаи
- [ ] Протестировано на реальных задачах

## Итоги

- Начинайте с валидации структуры через `quick_validate.py`
- Тестируйте каждый скрипт вручную
- Проверяйте навык в реальном Codex
- Итерируйте на основе опыта использования

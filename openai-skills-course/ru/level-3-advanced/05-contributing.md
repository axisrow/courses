# Контрибьюция в openai/skills

## Введение

Если вы создали полезный навык, вы можете предложить его в официальный репозиторий [openai/skills](https://github.com/openai/skills). В этом уроке мы разберём процесс контрибьюции.

## Ценности сообщества

Репозиторий openai/skills придерживается следующих принципов:

- **Будьте добры и инклюзивны** — уважайте других участников
- **Предполагайте добрые намерения** — письменная коммуникация сложна, толкуйте неоднозначности в лучшую сторону
- **Учите и учитесь** — если видите что-то непонятное, откройте issue или PR с улучшениями

## Процесс контрибьюции

### Шаг 1: Форкните репозиторий

Перейдите на [github.com/openai/skills](https://github.com/openai/skills) и нажмите «Fork».

### Шаг 2: Клонируйте форк

```bash
git clone https://github.com/ваш-логин/skills.git
cd skills
```

### Шаг 3: Создайте ветку

```bash
git checkout -b add-my-skill
```

### Шаг 4: Добавьте навык

Поместите ваш навык в соответствующую папку:

- `skills/.experimental/` — для новых, экспериментальных навыков
- `skills/.curated/` — только по приглашению или после проверки

```bash
cp -r ~/my-skill skills/.experimental/my-skill
```

### Шаг 5: Проверьте навык

```bash
python skills/.system/skill-creator/scripts/quick_validate.py skills/.experimental/my-skill
```

### Шаг 6: Создайте Pull Request

```bash
git add .
git commit -m "Add my-skill to experimental"
git push origin add-my-skill
```

Перейдите на GitHub и создайте Pull Request.

## Что включить в PR

- Описание навыка и его назначения
- Примеры использования
- Результаты тестирования

## Безопасность

Если вы обнаружили уязвимость или у вас есть вопросы по безопасности:

- Напишите на **security@openai.com**
- Не публикуйте уязвимости в открытых issues

## Путь навыка

```
Ваша идея → experimental → (проверка) → curated → (возможно) system
```

Экспериментальные навыки, которые показали себя хорошо, могут быть перенесены в курированные.

## Итоги

- Контрибьюция начинается с форка и Pull Request
- Новые навыки добавляются в `.experimental`
- Следуйте ценностям сообщества
- Хорошие экспериментальные навыки могут стать курированными
- Вопросы безопасности направляйте на security@openai.com

# Публикация навыков

## Введение

Вы создали и протестировали навык — теперь пора поделиться им с миром. В этом уроке мы разберём способы публикации навыков.

## Способ 1: GitHub-репозиторий

Самый простой способ — выложить навык в свой GitHub-репозиторий.

### Шаг 1: Создайте репозиторий

```bash
mkdir my-skills && cd my-skills
git init
```

### Шаг 2: Добавьте навык

```
my-skills/
└── csv-analyzer/
    ├── SKILL.md
    ├── agents/
    │   └── openai.yaml
    ├── scripts/
    │   └── analyze.py
    └── LICENSE.txt
```

### Шаг 3: Опубликуйте

```bash
git add .
git commit -m "Add csv-analyzer skill"
git remote add origin https://github.com/you/my-skills.git
git push -u origin main
```

### Шаг 4: Установка другими пользователями

Теперь любой может установить ваш навык:

```
$skill-installer install https://github.com/you/my-skills/tree/main/csv-analyzer
```

## Способ 2: Контрибьюция в openai/skills

Вы можете предложить навык в официальный репозиторий OpenAI. Подробнее об этом — в следующем уроке.

## Лицензия

Каждый навык должен содержать файл `LICENSE.txt` с лицензией. Это важно для:

- Ясности условий использования
- Защиты ваших прав
- Соответствия требованиям openai/skills

## Подготовка к публикации

### Чек-лист:

- [ ] Навык прошёл валидацию (`quick_validate.py`)
- [ ] Все скрипты протестированы
- [ ] SKILL.md содержит понятное описание
- [ ] `agents/openai.yaml` заполнен
- [ ] Добавлен `LICENSE.txt`
- [ ] Нет конфиденциальных данных (ключи, пароли, токены)
- [ ] Нет лишних файлов (README, CHANGELOG)

## Итоги

- Навыки публикуются через GitHub-репозиторий
- Skill-installer умеет устанавливать навыки из любого GitHub-репо
- Каждый навык должен содержать лицензию
- Перед публикацией проверьте навык на конфиденциальные данные

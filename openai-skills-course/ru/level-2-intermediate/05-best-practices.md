# Лучшие практики

## Введение

В этом уроке собраны рекомендации по созданию качественных навыков. Следуя этим правилам, вы сделаете навыки, которые работают надёжно и эффективно.

## 1. Краткость — ваш друг

Контекстное окно Codex ограничено. Каждый навык делит его с системным промптом, историей разговора и другими навыками.

**Правило:** Если Codex и так это знает — не пишите.

❌ Плохо:
```markdown
Python — это язык программирования, который используется для написания скриптов. Скрипты — это файлы с кодом...
```

✅ Хорошо:
```markdown
Запустить `scripts/process.py` для обработки данных.
```

## 2. Описание (description) решает всё

Description в frontmatter — это единственное, что Codex всегда видит. Если описание неточное, навык не будет активироваться.

**Включите в описание:**
- Что навык делает
- Конкретные триггеры (слова, сценарии)
- Типы файлов или инструментов

## 3. Прогрессивное раскрытие

Не загружайте всю информацию в SKILL.md. Используйте три уровня:

1. **Метаданные** (~100 слов) — всегда в контексте
2. **SKILL.md** (<5000 слов) — загружается при активации
3. **References** — загружается по необходимости

## 4. Степени свободы

Выбирайте уровень детализации в зависимости от задачи:

- **Высокая свобода** — текстовые инструкции, много вариантов
- **Средняя свобода** — псевдокод, основной паттерн с вариациями
- **Низкая свобода** — конкретные скрипты, строгая последовательность

**Правило:** Чем опаснее ошибка, тем ниже свобода.

## 5. Не создавайте лишних файлов

В навык НЕ нужно добавлять:
- README.md
- CHANGELOG.md
- INSTALLATION_GUIDE.md
- Документацию для пользователей

Навык читает только AI-агент, а не человек.

## 6. Тестируйте скрипты

Каждый скрипт в `scripts/` должен быть запущен и проверен перед публикацией. Убедитесь, что:
- Скрипт запускается без ошибок
- Выходные данные соответствуют ожиданиям
- Обработаны крайние случаи

## 7. Именование

- Только строчные буквы, цифры и дефисы
- Короткие, начинающиеся с глагола фразы
- Используйте пространства имён для инструментов: `gh-fix-ci`, `notion-meeting-intelligence`

## Чек-лист качественного навыка

- [ ] SKILL.md содержит name и description во frontmatter
- [ ] Description подробно описывает, когда использовать навык
- [ ] Тело SKILL.md краткое и практичное (до 500 строк)
- [ ] Скрипты протестированы
- [ ] Нет лишних файлов (README, CHANGELOG)
- [ ] Имя навыка в kebab-case
- [ ] `agents/openai.yaml` заполнен

## Итоги

- Пишите кратко — контекстное окно ограничено
- Description — главный триггер навыка
- Используйте прогрессивное раскрытие
- Не создавайте лишних файлов
- Тестируйте все скрипты

# Тестирование навыков

## Введение

Навык работает настолько хорошо, насколько хорошо он протестирован. В этом уроке — методы проверки навыков.

## Ручное тестирование

### Шаг 1: Проверьте активацию

Убедитесь, что навык активируется при правильных запросах:

```
✅ «Проверь мой Python-код» → навык code-reviewer активируется
✅ «Сделай ревью этого кода» → активируется
❌ «Расскажи о Python» → НЕ должен активироваться
```

### Шаг 2: Проверьте инструкции

Дайте Claude типовую задачу и проверьте:
- Следует ли Claude указанному процессу?
- Использует ли правильный формат вывода?
- Соблюдает ли ограничения?

### Шаг 3: Граничные случаи

Проверьте нестандартные ситуации:
- Пустой ввод
- Очень большой ввод
- Некорректные данные
- Запросы на другом языке

## Чек-лист тестирования

- [ ] Frontmatter валиден (YAML без ошибок)
- [ ] `name` уникален и в правильном формате
- [ ] `description` содержит триггеры
- [ ] Инструкции понятны и однозначны
- [ ] Примеры корректны
- [ ] Навык не конфликтует с другими
- [ ] Скрипты (если есть) работают без ошибок

## Тестирование скриптов

Если навык содержит скрипты, тестируйте их отдельно:

```bash
# Пример тестирования скрипта валидации
python scripts/office/validate.py test-document.docx
```

## Итерации

Тестирование навыков — итеративный процесс:

1. Создайте навык
2. Протестируйте с 5-10 запросами
3. Найдите проблемы
4. Уточните инструкции
5. Повторите

## Итоги

- Тестируйте активацию, следование инструкциям и граничные случаи
- Используйте чек-лист для систематической проверки
- Тестирование — итеративный процесс
- Скрипты тестируйте отдельно

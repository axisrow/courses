# Сложные навыки со скриптами

## Введение

Некоторые задачи требуют не только инструкций, но и скриптов, шаблонов и ресурсов. Рассмотрим создание сложных навыков.

## Структура сложного навыка

```
webapp-testing/
├── SKILL.md
├── scripts/
│   └── with_server.py       # Скрипт запуска сервера
├── examples/
│   ├── static_html_automation.py
│   ├── element_discovery.py
│   └── console_logging.py
└── LICENSE.txt
```

## Скрипты

Скрипты расширяют возможности навыка:

```python
# scripts/with_server.py — пример из webapp-testing
# Скрипт запускает локальный сервер для тестирования
```

Claude может вызывать скрипты навыка для выполнения операций, которые невозможны в чистом тексте.

## Ассеты и ресурсы

### Справочные материалы

```
mcp-builder/
├── SKILL.md
└── reference/
    ├── evaluation.md
    ├── python_mcp_server.md
    └── mcp_best_practices.md
```

Навык `mcp-builder` содержит подробную документацию в папке `reference/`, на которую ссылается `SKILL.md`.

### Шаблоны и архивы

```
web-artifacts-builder/
├── SKILL.md
└── scripts/
    ├── shadcn-components.tar.gz   # Архив с компонентами
    ├── init-artifact.sh           # Скрипт инициализации
    └── bundle-artifact.sh         # Скрипт сборки
```

## XML-схемы в документных навыках

Навык `docx` включает XML-схемы Office для валидации документов:

```
docx/scripts/office/schemas/
├── ISO-IEC29500-4_2016/    # Стандарт OOXML
├── ecma/                    # ECMA стандарты
├── mce/                     # MCE
└── microsoft/               # Расширения Microsoft
```

## Рекомендации

1. **Скрипты** — для обработки бинарных данных и сложной логики
2. **Reference** — для объёмной документации, которая не помещается в SKILL.md
3. **Шаблоны** — для стандартизации вывода
4. **Схемы** — для валидации результатов

## Итоги

- Сложные навыки включают скрипты, ассеты и справочные материалы
- Скрипты расширяют возможности за пределы текстовых инструкций
- Справочные материалы размещаются в папке `reference/`
- XML-схемы и шаблоны обеспечивают валидацию

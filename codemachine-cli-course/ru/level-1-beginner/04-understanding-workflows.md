# Понимание Workflow

## Что такое workflow?

Workflow — это последовательность шагов, которые говорят AI-агентам, что делать и в каком порядке. Каждый шаг включает:

- **Агента** — какой AI выполняет задачу
- **Промпт** — инструкции для агента
- **Настройки** — как агент должен себя вести

## Пример из жизни

Допустим, вы хотите создать новую фичу для сайта:

1. Исследовать требования
2. Спланировать архитектуру
3. Написать код
4. Проревьюить код
5. Написать тесты

В CodeMachine каждый пункт становится **шагом** со своим агентом и инструкциями.

## Файл workflow

```javascript
export default {
  name: 'Feature Builder',
  steps: [
    resolveStep('research-agent'),
    resolveStep('planning-agent'),
    resolveStep('coding-agent'),
    resolveStep('review-agent'),
    resolveStep('testing-agent'),
  ],
};
```

## Три типа строительных блоков

| Блок | Назначение | Пример |
|------|-----------|--------|
| **Step** (шаг) | Одна задача | «Проревьюить код» |
| **Module** (модуль) | Задача с циклом | «Проверять и исправлять» |
| **Folder** (папка) | Группа шагов | «Все шаги настройки» |

## Как шаги связаны

Шаги выполняются последовательно:

```
Шаг 1 → Шаг 2 → Шаг 3 → Шаг 4 → Готово!
```

## Итог

- Workflow — сохранённая последовательность AI-задач
- Каждый шаг имеет агента, промпт и настройки
- Шаги, модули и папки — строительные блоки

# Понимание агентов

## Что такое агент?

Агент — это AI-ассистент, назначенный на конкретную задачу. У каждого агента есть:

- **ID** — уникальное имя (например, `code-reviewer`)
- **Отображаемое имя** — понятное название (например, «Старший ревьюер кода»)
- **Промпт** — подробные инструкции
- **Движок** — какая AI-система используется (Claude, Codex и т.д.)

## Типы агентов

### 1. Основные агенты (Main Agents)
Главные исполнители. Каждый шаг workflow использует основного агента.

```javascript
{
  id: 'code-reviewer',
  name: 'Code Reviewer',
  description: 'Проверяет качество кода',
  promptPath: 'prompts/review.md',
}
```

### 2. Суб-агенты (Sub-Agents)
Помощники, работающие параллельно с основными агентами.

```javascript
{
  id: 'frontend-dev',
  name: 'Frontend Developer',
  description: 'Отвечает за UI',
  mirrorPath: 'prompts/frontend-mirror.md',
}
```

### 3. Контроллер (Controller)
Специальный агент, который наблюдает за всем workflow и принимает решения.

## Выбор движка

| Движок | Лучше всего для |
|--------|----------------|
| Claude | Анализ, ревью, документация |
| Codex | Генерация кода, реализация |
| Cursor | Задачи в IDE |

Можно комбинировать движки:

```javascript
resolveStep('planning', { engine: 'claude' }),
resolveStep('coding', { engine: 'codex' }),
resolveStep('review', { engine: 'claude' }),
```

## Итог

- Агенты — AI-работники в workflow
- Основные агенты, суб-агенты и контроллеры
- У каждого агента — свой ID, имя, промпт и движок
- Можно комбинировать разные AI-движки

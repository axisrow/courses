# Ваш первый проект

## Создание папки проекта

Создадим простой проект для практики:

```bash
mkdir my-first-workflow
cd my-first-workflow
```

## Запуск CodeMachine

```bash
codemachine
```

При первом запуске CodeMachine создаёт папку `.codemachine` внутри проекта — в ней хранятся все настройки.

## Папка .codemachine

```
my-first-workflow/
└── .codemachine/
    ├── config/          ← настройки агентов и workflow
    ├── prompts/         ← инструкции для AI-агентов
    ├── inputs/          ← спецификации проекта
    └── logs/            ← логи выполнения
```

## Текстовый интерфейс (TUI)

При запуске вы видите текстовый интерфейс в терминале. Он показывает:

- Статус текущего workflow
- Какой агент активен
- Таймлайн выполненных шагов

## Горячие клавиши

| Клавиша | Действие |
|---------|----------|
| Tab | Показать/скрыть панель таймлайна |
| Shift+Tab | Переключить автономный режим |
| P | Поставить на паузу |
| H | Просмотр истории |
| ↑ / ↓ | Навигация |
| Escape | Остановка (с подтверждением) |

## Workflow-билдер Ali

CodeMachine включает встроенный workflow под названием **Ali**. Ali — дружелюбный помощник, который создаёт ваш первый workflow, задавая вопросы и генерируя файлы.

## Итог

- Создайте папку проекта и запустите `codemachine`
- Папка `.codemachine` создаётся автоматически
- TUI — визуальный интерфейс в терминале
- Ali поможет создать первый workflow

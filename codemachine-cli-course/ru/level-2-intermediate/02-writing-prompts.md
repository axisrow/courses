# Написание эффективных промптов

## Структура файла промпта

```markdown
---
name: 'Название шага'
description: 'Что делает этот шаг'
---

# Заголовок шага

## ЦЕЛЬ ШАГА:
[Чёткая цель в одном предложении]

## ОБЯЗАТЕЛЬНЫЕ ПРАВИЛА:
[Правила, которым агент должен следовать]

## Последовательность инструкций:
### 1. Первая задача
### 2. Вторая задача
### 3. Третья задача

## МЕТРИКИ УСПЕХА/НЕУДАЧИ:
### УСПЕХ: [Как выглядит «готово»]
### НЕУДАЧА: [Что считается провалом]
```

## Почему структура важна

Без структуры AI-агенты:
- Пропускают шаги
- Делают предположения
- Дают непоследовательные результаты

## Советы по написанию

### Будьте конкретны
❌ «Проревьюй код»
✅ «Проревьюй код в папке `src/`. Проверь на уязвимости, проблемы производительности и стиль кода.»

### Устанавливайте границы
```markdown
## ГРАНИЦЫ КОНТЕКСТА:
- НЕ модифицировать файлы
- НЕ менять схему базы данных
- Фокус ТОЛЬКО на frontend-компонентах
```

### Используйте пронумерованные инструкции
```markdown
### 1. Прочитать спецификацию
### 2. Перечислить все API-эндпоинты
### 3. Для каждого задокументировать вход и выход
### 4. Сохранить отчёт в output/api-summary.md
```

## Цепочки промптов

Для сложных шагов разбейте инструкции на несколько файлов:

```javascript
{
  id: 'planner',
  chainedPromptsPath: [
    'prompts/plan/step-01.md',
    'prompts/plan/step-02.md',
    'prompts/plan/step-03.md',
  ],
}
```

## Итог

- Следуйте стандартной структуре промпта
- Будьте конкретны и ставьте границы
- Нумеруйте инструкции
- Определяйте метрики успеха
- Используйте цепочки для сложных шагов

# 编写有效的提示词

## 提示词文件结构

```markdown
---
name: '步骤名称'
description: '这个步骤做什么'
---

# 步骤标题

## 步骤目标：
[清晰的一句话目标]

## 强制执行规则：
[代理必须遵循的规则]

## 指令序列：
### 1. 第一个任务
### 2. 第二个任务
### 3. 第三个任务

## 成功/失败指标：
### 成功：[什么算"完成"]
### 失败：[什么算失败]
```

## 为什么结构很重要

没有结构，AI 代理会：
- 跳过步骤
- 做出假设
- 产生不一致的结果

## 编写技巧

### 要具体
❌ "审查代码"
✅ "审查 `src/` 文件夹中的代码。检查安全漏洞、性能问题和代码风格。"

### 设定边界
```markdown
## 上下文边界：
- 不要修改任何文件
- 不要更改数据库架构
- 仅关注前端组件
```

### 使用编号指令
```markdown
### 1. 阅读规格说明文件
### 2. 列出所有 API 端点
### 3. 为每个端点记录输入和输出
### 4. 保存摘要到 output/api-summary.md
```

## 链式提示词

```javascript
{
  id: 'planner',
  chainedPromptsPath: [
    'prompts/plan/step-01.md',
    'prompts/plan/step-02.md',
    'prompts/plan/step-03.md',
  ],
}
```

## 总结

- 遵循标准提示词结构
- 要具体，设定边界
- 使用编号指令
- 定义成功和失败指标
- 复杂步骤使用链式提示词

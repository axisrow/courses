# Аутентификация

Добавьте вход и регистрацию в приложение.

## Supabase Auth

Lovable использует Supabase Auth:

- Email + пароль
- Вход через Google
- Вход через GitHub
- Magic link (без пароля)

## Добавление авторизации

> Добавь аутентификацию. Пользователи регистрируются по email и паролю и входят в систему.

Lovable создаст:
- Страницу регистрации
- Страницу входа
- Управление сессиями
- Защищённые маршруты

## Вход через соцсети

> Добавь вход через Google на странице регистрации.

Потребуется настроить Google OAuth в Supabase.

## Защищённые страницы

> Дашборд доступен только авторизованным. Остальных перенаправляй на страницу входа.

## Профили пользователей

> После регистрации пользователь заполняет профиль: имя, аватар, био. Храни в таблице profiles.

## Отображение пользователя

> Покажи имя и аватар пользователя в правом верхнем углу. Добавь кнопку выхода.

## Поток авторизации

```
Пользователь заходит
  → Не авторизован → Страница входа
  → Авторизован → Дашборд
    → Выход → Обратно на вход
```

## Типичные паттерны

| Паттерн | Описание |
|---------|----------|
| Роли | Админ vs обычный пользователь |
| Подтверждение email | Проверка почты перед доступом |
| Сброс пароля | «Забыли пароль?» |
| Сохранение сессии | Оставаться в системе |

## Итог

Supabase Auth берёт на себя вход, регистрацию и сессии. Lovable настраивает всё через промпты.

# Оптимизация производительности

Сделайте приложение быстрым.

## Ленивая загрузка

> Загружай страницу дашборда лениво — только при переходе на неё.

Это уменьшает начальный размер бандла.

## Оптимизация изображений

> Используй ленивую загрузку для изображений. Задай ширину и высоту.

## Пагинация

> Вместо загрузки всех 1000 постов, добавь пагинацию — 20 постов на страницу.

## Кэширование

> Кэшируй ответ API на 5 минут.

## Разделение кода

Lovable использует `lazy()` и `Suspense` для автоматического разделения.

> Выдели админ-раздел в отдельный чанк.

## Оптимизация запросов к БД

> Добавь индекс на колонку «created_at». Используй пагинацию в запросе Supabase.

## Метрики

| Метрика | Цель |
|---------|------|
| First Contentful Paint | < 1.5с |
| Largest Contentful Paint | < 2.5с |
| Размер бандла | < 500КБ |
| Время ответа API | < 200мс |

## Дебаунсинг

> Добавь дебаунс для поиска — ожидание 300мс перед запросом.

## Итог

Оптимизируйте ленивой загрузкой, пагинацией, кэшированием и разделением кода.

# 性能优化

让你的应用更快。

## 懒加载

> 仪表盘页面使用懒加载——只在用户导航到时才加载。

这减少了初始包大小。

## 图片优化

> 图片使用懒加载。设置正确的宽度和高度。

## 分页

> 不要一次加载全部 1000 篇文章，添加分页——每页 20 篇。

## 缓存

> 将 API 响应缓存 5 分钟。

## 代码分割

Lovable 使用 React 的 `lazy()` 和 `Suspense` 自动代码分割。

> 将管理员部分分割成单独的代码块。

## 数据库查询优化

> 在「created_at」列添加索引。在 Supabase 查询中使用分页。

## 性能指标

| 指标 | 目标 |
|------|------|
| First Contentful Paint | < 1.5秒 |
| Largest Contentful Paint | < 2.5秒 |
| 总包大小 | < 500KB |
| API 响应时间 | < 200毫秒 |

## 防抖

> 搜索输入添加防抖——等待 300 毫秒再发送请求。

## 总结

通过懒加载、分页、缓存和代码分割来优化。通过提示词让 Lovable 实现这些模式。

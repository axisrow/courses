# 团队协作

与团队一起在 Lovable 项目上工作。

## 邀请成员

通过 **Settings → Team** 邀请团队成员。

## 角色

| 角色 | 权限 |
|------|------|
| Owner | 完全访问、账单、删除项目 |
| Editor | 编辑代码、使用提示词、发布 |
| Viewer | 仅查看 |

## 协作方式

- 每个成员都可以独立发送提示词。
- 更改实时同步。
- 聊天记录显示谁做了什么更改。

## 版本历史

Lovable 保留所有更改的历史：

- 查看以前的版本。
- 如果出问题可以回滚。
- 查看每个提示词的更改内容。

## 最佳实践

1. **先统一结构。** 确定页面、组件和命名。
2. **写清楚的提示词。** 其他人应该能理解你的意图。
3. **发布前检查。** 互相审查更改。
4. **不要同时发提示词。** 轮流操作避免冲突。

## 总结

Lovable 支持团队协作，包括角色管理、版本历史和实时同步。与团队协调以获得最佳效果。

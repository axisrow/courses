# 高级 Cascade 工作流

## 超越简单提示

在这个级别，你可以用 Cascade 处理复杂的多步骤任务。

## 链式提示

将大任务分解为步骤：

1. "分析当前项目结构并建议改进"
2. "实施你建议的重构"
3. "更新所有导入路径以匹配新结构"

每个提示都建立在之前的对话上下文上。

## 用 Cascade 进行系统设计

先设计再编码：

```
我需要一个通知系统，支持邮件、短信和推送通知。
先设计类层次结构和接口，然后实现。
```

## 代码生成模式

### 从规范生成
```
这是我的 API 规范（OpenAPI）：
[粘贴规范]
生成 Express 路由和控制器。
```

### 从示例生成
```
输入：[1, 2, 3, 4, 5]
输出：[2, 4]（仅偶数）
编写通用过滤函数。
```

### 从代码生成测试
```
查看 src/services/auth.js，生成全面的单元测试，覆盖边界情况。
```

## 多轮对话

持续对话以优化结果：

1. "为 API 创建缓存层"
2. "添加 TTL 支持"
3. "改用 Redis 替代内存缓存"
4. "数据更新时添加缓存失效"

## 使用 @提及

在 Cascade 中可以引用：
- `@file` — 指定文件
- `@folder` — 引用目录
- `@symbol` — 引用函数或类

## 练习

用链式提示从零设计和实现一个完整功能。

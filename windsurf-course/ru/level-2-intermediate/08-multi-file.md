# Работа с несколькими файлами

## Зачем?

В реальных проектах много файлов. Добавление фичи часто требует изменений в нескольких местах. Windsurf это понимает.

## Как Cascade работает с файлами

Cascade может:

1. **Читать** связанные файлы для понимания контекста
2. **Редактировать** несколько файлов за один раз
3. **Создавать** новые файлы
4. **Показывать diff** для каждого файла отдельно

## Пример: новый API-эндпоинт

Запрос к Cascade:
```
Добавь GET /api/users/:id, который возвращает данные пользователя из базы
```

Cascade может изменить:
- `routes/users.js` — добавить маршрут
- `controllers/userController.js` — добавить обработчик
- `models/user.js` — проверить модель
- `tests/users.test.js` — добавить тест

## Просмотр изменений

1. Cascade показывает список изменённых файлов
2. Нажмите на файл для просмотра diff
3. Примите или отклоните изменения по файлам
4. Можно принять всё сразу

## Советы

- **Упоминайте архитектуру**: «Используя MVC, добавь...»
- **Ссылайтесь на файлы**: «Обнови `config.js` и `server.js`...»
- **Описывайте полную фичу**: чем больше контекста, тем лучше

## Ограничения

- Cascade читает открытые и соседние файлы
- В очень больших проектах укажите конкретные папки
- Бинарные файлы (картинки) пропускаются

## Практика

Попросите Cascade добавить полную фичу (например, регистрацию) и понаблюдайте за изменениями в нескольких файлах.

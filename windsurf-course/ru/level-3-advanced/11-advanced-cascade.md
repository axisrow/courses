# Продвинутые сценарии Cascade

## Сложные задачи

На этом уровне Cascade решает многошаговые задачи.

## Цепочки промптов

Разбивайте большие задачи на шаги:

1. «Проанализируй структуру проекта и предложи улучшения»
2. «Реализуй предложенную реструктуризацию»
3. «Обнови все пути импортов под новую структуру»

Каждый промпт использует контекст предыдущего разговора.

## Проектирование с Cascade

Попросите спроектировать перед кодированием:

```
Мне нужна система уведомлений: email, SMS, push.
Сначала спроектируй иерархию классов и интерфейсы, потом реализуй.
```

## Паттерны генерации кода

### По спецификации
```
Вот мой API-спек (OpenAPI):
[вставьте спек]
Сгенерируй маршруты и контроллеры Express.
```

### По примерам
```
Вход: [1, 2, 3, 4, 5]
Выход: [2, 4] (только чётные)
Напиши универсальную функцию фильтрации.
```

### Тесты по коду
```
Посмотри src/services/auth.js и сгенерируй полные юнит-тесты
с крайними случаями.
```

## Многоходовые диалоги

1. «Создай слой кэширования для API»
2. «Добавь поддержку TTL»
3. «Переключи на Redis вместо in-memory»
4. «Добавь инвалидацию кэша при обновлении данных»

## Использование @-упоминаний

В Cascade можно ссылаться на:
- `@file` — конкретный файл
- `@folder` — директорию
- `@symbol` — функцию или класс

## Практика

Спроектируйте и реализуйте полную фичу с помощью цепочки промптов.

# Векторный поиск vs. поиск на основе рассуждений

## Два подхода к поиску информации

Есть два принципиально разных способа, которыми компьютер может искать информацию в документе:

### Подход 1: Векторный поиск (традиционный)

1. Разбить документ на маленькие кусочки (обычно 200-500 слов)
2. Превратить каждый кусочек в список чисел («вектор»)
3. Когда пользователь задаёт вопрос, превратить вопрос тоже в числа
4. Найти кусочки, чьи числа ближе всего к числам вопроса
5. Передать эти кусочки ИИ для генерации ответа

### Подход 2: Поиск на основе рассуждений (PageIndex)

1. Проанализировать весь документ, понять его структуру
2. Построить иерархическое дерево с названиями разделов, резюме и диапазонами страниц
3. Когда пользователь задаёт вопрос, использовать ИИ для рассуждения по дереву
4. Перейти к наиболее релевантным разделам на основе понимания
5. Извлечь полный контекст из этих разделов

## Сравнение двух подходов

| Параметр | Векторный поиск | PageIndex |
|----------|----------------|-----------|
| **Как ищет** | Математическая похожесть | Рассуждения ИИ |
| **Структура документа** | Игнорируется | Сохраняется |
| **Контекст** | Теряется | Сохраняется |
| **Нужна база данных** | Да (векторная БД) | Нет |
| **Нужна нарезка** | Да | Нет |
| **Объяснимость** | Низкая | Высокая |

## Когда векторный поиск буксует

- **Таблицы и числа**: «Выручка 5,2 млрд $» и «Общий доход составил 5 200 млн» — разные слова, одно значение
- **Отрицание**: «Компания НЕ сообщала об убытках» и «Компания сообщала об убытках» — для вектора почти одинаковые
- **Многошаговое рассуждение**: Когда нужно связать информацию из разных разделов

## Простая аналогия

- **Векторный поиск** — это как искать в библиотеке, просматривая каждую страницу каждой книги на совпадение слов.
- **PageIndex** — это как спросить библиотекаря, который прочёл все книги и может рассудить, где найти нужное.

## Итоги

Векторный поиск опирается на математическую похожесть, которая часто путает «похожее» с «релевантным». PageIndex использует рассуждения ИИ по структуре документа, достигая более высокой точности — особенно для сложных профессиональных документов.

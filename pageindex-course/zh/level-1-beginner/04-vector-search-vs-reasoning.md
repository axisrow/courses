# 向量搜索与推理检索的对比

## 两种查找信息的方法

计算机在文档中查找信息，基本上有两种方式：

### 方法一：向量搜索（传统方法）

1. 将文档切成小块（通常 200-500 个词）
2. 用数学模型将每块转为一组数字（"向量"）
3. 用户提问时，也将问题转为数字
4. 找到数字最接近的文档块
5. 将这些块交给 AI 生成回答

### 方法二：推理检索（PageIndex）

1. 分析整个文档，理解它的结构
2. 构建一棵层级树，包含章节标题、摘要和页码范围
3. 用户提问时，AI 通过推理在树中导航
4. 根据理解定位到最相关的章节
5. 从该章节提取完整内容

## 两种方法的对比

| 特点 | 向量搜索 | PageIndex |
|------|---------|-----------|
| **查找方式** | 数学相似度 | AI 推理 |
| **文档结构** | 被忽略（块是独立的） | 被保留（层级树） |
| **上下文** | 丢失（小块） | 保留（完整章节） |
| **需要数据库** | 是（向量数据库） | 否 |
| **需要切块** | 是 | 否 |
| **可解释性** | 低 | 高（"答案来自第3.2节"） |

## 向量搜索的局限

向量搜索在以下情况容易出问题：

- **表格和数字**："52亿美元收入" 和 "总收入为5200百万" 用词不同但含义相同
- **否定句**："公司没有亏损" 和 "公司亏损了" 的向量非常相似
- **多步推理**：需要理解不同章节之间关系的信息

## PageIndex 的优势场景

PageIndex 在以下场景表现出色：

- 长篇且结构清晰的文档（报告、手册、教科书）
- 需要理解上下文的问题
- 准确性比速度更重要
- 需要说明答案的来源

## 一个简单的比喻

- **向量搜索** 就像在图书馆逐页扫描每本书来找匹配的词。
- **PageIndex** 就像问一位读过所有书的图书管理员，他能推理出在哪里找到您需要的内容。

## 总结

向量搜索依赖数学相似度，常常把"相似"和"相关"混淆。PageIndex 使用 AI 对文档结构进行推理，在复杂的专业文档中准确性更高。

# 树形结构详解

## 什么是树形结构？

树形结构是一种分层组织信息的方式，从概括到具体。你已经熟悉很多树形结构：

- **公司组织架构**：CEO → 部门 → 团队 → 个人
- **电脑文件系统**：文件夹 → 子文件夹 → 文件
- **书籍目录**：部分 → 章 → 节 → 小节

PageIndex 为任何文档创建这种结构。

## PageIndex 树的样子

以年度报告为例的简化树形结构：

```
年度报告 2023
├── 致股东信 (第1-3页)
├── 公司概览 (第4-10页)
│   ├── 我们的使命 (第4-5页)
│   ├── 核心产品 (第6-8页)
│   └── 全球布局 (第9-10页)
├── 财务报表 (第11-50页)
│   ├── 利润表 (第11-20页)
│   ├── 资产负债表 (第21-35页)
│   └── 现金流量表 (第36-50页)
└── 风险因素 (第51-60页)
```

## JSON 格式

PageIndex 内部以 JSON 格式存储树。每个章节是一个**节点**（node），包含：

- **title**：章节标题
- **node_id**：唯一标识符
- **start_index**：章节首页
- **end_index**：章节末页
- **summary**：内容简要描述
- **nodes**：子章节（子节点）

```json
{
  "title": "财务报表",
  "node_id": "0003",
  "start_index": 11,
  "end_index": 50,
  "summary": "2023财年完整财务报表",
  "nodes": [
    {
      "title": "利润表",
      "node_id": "0004",
      "start_index": 11,
      "end_index": 20
    }
  ]
}
```

## 为什么树形结构如此强大

树形结构让 PageIndex 能够：

1. **从宽到窄** —— 像地图一样：国家 → 城市 → 街道
2. **跳过无关章节** —— 不需要搜索全部300页
3. **保持上下文** —— 每个节点知道自己在整体中的位置

## 总结

树形结构是 PageIndex 的核心。它将文档组织成带页码引用的章节层级，让 AI 能像人类专家使用详细目录一样高效导航。

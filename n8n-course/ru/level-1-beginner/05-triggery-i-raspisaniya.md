# Триггеры и расписания

## Что такое триггеры?

Триггер — первый нод в workflow. Он определяет, **когда** workflow запускается.

## Виды триггеров

### Manual Trigger
Запуск по кнопке. Хорош для тестирования.

### Schedule Trigger
Запуск по таймеру:
- Каждые 5 минут
- Каждый день в 9:00
- Каждый понедельник
- Первого числа каждого месяца

**Примеры cron:**
```
0 9 * * *       → Каждый день в 9:00
0 9 * * 1       → Каждый понедельник в 9:00
*/30 * * * *    → Каждые 30 минут
```

### Webhook
Запуск при получении HTTP-запроса от внешнего сервиса.

1. Добавьте нод **Webhook**
2. Скопируйте URL
3. Вставьте URL в настройки внешнего сервиса

**Применение:** приём форм, уведомления от GitHub, Stripe и т.д.

### Триггеры приложений
- **Gmail Trigger** — новое письмо
- **Slack Trigger** — новое сообщение
- **GitHub Trigger** — новый issue

## Активация workflow

Триггеры работают только при **активном** workflow:

1. Постройте workflow
2. Включите переключатель **Active** (вверху справа)
3. Workflow теперь запускается автоматически

## Polling vs Webhook

| Особенность | Polling | Webhook |
|-------------|---------|---------|
| Как работает | n8n проверяет периодически | Сервис отправляет данные |
| Скорость | Небольшая задержка | Мгновенно |
| Настройка | Проще | Нужен URL |

## Практика

1. Создайте workflow с **Schedule Trigger** (каждую минуту)
2. Добавьте **HTTP Request** для получения времени: `http://worldtimeapi.org/api/timezone/UTC`
3. Активируйте и проверьте историю выполнений

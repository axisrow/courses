# Основные ноды

## Что такое ноды?

Ноды — строительные блоки workflow. Каждый нод выполняет одну задачу. Соединяя ноды, вы создаёте мощные автоматизации.

## Категории нодов

### Триггерные ноды
Запускают workflow:
- **Manual Trigger** — по нажатию кнопки
- **Schedule Trigger** — по расписанию
- **Webhook** — при получении HTTP-запроса

### Ноды действий
Выполняют работу:
- **HTTP Request** — вызов любого API
- **Set** — создание и изменение полей
- **IF** — принятие решений
- **Code** — собственный JavaScript или Python
- **Merge** — объединение данных

### Ноды приложений
Подключение к конкретным сервисам:
- **Gmail** — отправка и чтение писем
- **Slack** — сообщения в каналы
- **Google Sheets** — работа с таблицами
- **Notion** — страницы и базы данных
- **GitHub** — репозитории и задачи

## Важнейшие ноды

### HTTP Request
Самый универсальный нод. Подключается к любому API.

### Set
Преобразование данных: переименование полей, добавление новых, удаление лишних.

### IF
Два пути на основе условия:
- **True** — условие выполнено
- **False** — не выполнено

Пример: если `статус` = «срочно» → алерт в Slack, иначе → запись в таблицу.

## Как добавить нод

1. Нажмите `+`
2. Найдите нод по имени или категории
3. Настройте параметры
4. Нажмите **Execute Step** для теста

## Практика

Постройте: **Manual Trigger** → **Set** (имя и возраст) → **IF** (возраст > 18) → два **Set** (один «взрослый», другой «ребёнок»).

## Итог

Вы знаете основные типы нодов. Далее — триггеры и расписания.

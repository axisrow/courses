# Первый запуск

## Введение

Вы установили OMP и настроили API-ключ. Давайте запустим агента и попробуем основные функции.

## Запуск

```bash
omp
```

Вы увидите экран приветствия с логотипом и подсказками.

## Настройка ролей моделей

Используйте `/model` для назначения моделей:

- **default** — обычная работа
- **smol** — быстрые/дешёвые задачи
- **slow** — глубокий анализ
- **plan** — планирование
- **commit** — генерация коммитов

## Первые действия

### Поговорите с агентом

```
> Какие файлы есть в текущей директории?
```

### Используйте @ для ссылки на файлы

```
> Объясни @src/main.ts
```

### Выполните команду

```
> !git status
```

(Префикс `!` выполняет команду и включает вывод в контекст.)

### Попробуйте режим планирования

```
/plan
> Спланируй рефакторинг модуля авторизации
```

## Управление сессиями

```bash
omp -c              # Продолжить последнюю сессию
omp -r              # Выбрать сессию
omp --no-session    # Без сохранения
```

## Полезные слэш-команды

| Команда | Описание |
|---------|----------|
| `/model` | Выбор модели |
| `/plan` | Режим планирования |
| `/settings` | Настройки |
| `/extensions` | Расширения и контекст |
| `/compact` | Сжать контекст |
| `/export` | Экспорт в HTML |
| `/exit` | Выход |

## Итоги

Вы запустили OMP, настроили модель и попробовали основные функции. В следующем уровне изучим продвинутые инструменты.

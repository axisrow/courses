# Понимание агентов

## Что такое агент?

Агент — это AI-работник с чёткой идентичностью. У каждого агента есть:

- **Role (Роль)** — чем занимается агент
- **Goal (Цель)** — чего агент стремится достичь
- **Backstory (Предыстория)** — контекст, формирующий поведение

## Создание агента

```python
from crewai import Agent

writer = Agent(
    role="Блогер",
    goal="Писать увлекательные статьи о технологиях",
    backstory="Вы — опытный техноблогер с 10-летним стажем.",
    verbose=True
)
```

## Основные параметры

| Параметр | Тип | Описание |
|----------|-----|----------|
| `role` | str | Должность агента |
| `goal` | str | Цель агента |
| `backstory` | str | Предыстория и контекст |
| `verbose` | bool | Подробные логи |
| `allow_delegation` | bool | Может ли делегировать задачи |
| `tools` | list | Доступные инструменты |
| `llm` | str | Какую языковую модель использовать |
| `max_iter` | int | Максимум итераций рассуждений |
| `memory` | bool | Включить память |

## Несколько агентов

```python
researcher = Agent(
    role="Исследователь",
    goal="Найти точные данные о рыночных трендах",
    backstory="Эксперт-аналитик рынка."
)

writer = Agent(
    role="Автор",
    goal="Превращать исследования в понятные отчёты",
    backstory="Технический писатель, который упрощает сложные темы."
)
```

## Советы

1. **Будьте конкретны** в ролях — «SEO-копирайтер» лучше, чем «Автор»
2. **Делайте цели измеримыми** — «Написать резюме на 500 слов» лучше, чем «Написать что-то»
3. **Используйте предысторию** для задания тона и уровня экспертизы
4. **Начинайте с verbose=True** для понимания работы

## YAML-конфигурация

```yaml
researcher:
  role: "Старший исследователь"
  goal: "Находить полную информацию по заданным темам"
  backstory: "Вы — тщательный исследователь, который всегда проверяет факты."
```

## Итого

Агенты — это работники вашей команды. Определяйте их с чёткой ролью, целью и предысторией. Чем конкретнее — тем лучше результат.

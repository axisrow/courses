# Память и контекст

## Зачем нужна память?

По умолчанию каждый запуск команды начинается с нуля. Память позволяет агентам запоминать информацию между задачами и даже между запусками.

## Типы памяти

| Тип | Область | Назначение |
|-----|---------|-----------|
| **Short-term** | Текущий запуск | Обмен данными между задачами |
| **Long-term** | Между запусками | Обучение на прошлом опыте |
| **Entity** | Между запусками | Запоминание фактов о сущностях |

## Включение памяти

```python
crew = Crew(
    agents=[researcher, writer],
    tasks=[research_task, writing_task],
    memory=True,
    verbose=True
)
```

## Как это работает

1. **Краткосрочная память** — хранит результаты задач во время запуска
2. **Долгосрочная память** — сохраняет ключевые выводы после завершения
3. **Память сущностей** — отслеживает людей, компании, концепции

## Контекст между задачами

Даже без памяти можно передавать контекст явно:

```python
writing_task = Task(
    description="Напишите статью на основе исследования.",
    expected_output="Статья на 500 слов.",
    agent=writer,
    context=[research_task]
)
```

## Когда использовать

- **Краткосрочная**: всегда полезна в многозадачных командах
- **Долгосрочная**: когда запускаете одну и ту же команду повторно
- **Сущностей**: когда работаете с повторяющимися объектами (клиенты, продукты)

## Итого

Память позволяет командам учиться и запоминать. Включайте через `memory=True`. Используйте context для явной передачи данных между задачами.

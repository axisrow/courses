# Пользовательские правила и .cursorrules

## Что такое пользовательские правила?

Вы можете указать Cursor, **как** писать код для вашего проекта. Правила гарантируют, что AI следует вашему стилю и соглашениям.

## Файл .cursorrules

Создайте файл `.cursorrules` в корне проекта. Он содержит инструкции, которым AI будет следовать при каждом взаимодействии.

### Пример

```
Ты помогаешь с проектом на React и TypeScript.

Правила:
- Используй функциональные компоненты с хуками (без классов)
- TypeScript в строгом режиме
- Пиши тесты для каждой новой функции (Vitest)
- Стили через Tailwind CSS
- Следуй структуре папок в src/
- Добавляй JSDoc-комментарии к экспортируемым функциям
- Обрабатывай ошибки через try/catch
- Комментарии к коду на русском
```

## Глобальные vs. проектные правила

- **Проектные** (`.cursorrules`) — только для одного проекта
- **Глобальные** (Settings → Cursor → Rules) — для всех проектов

## Что включить в правила

- Предпочтения по **языку и фреймворку**
- **Стиль кода** (именование, форматирование)
- Требования к **тестам**
- Стандарты **документации**
- Паттерны **обработки ошибок**
- Что **избегать** (устаревшие API, определённые библиотеки)

## Советы

- Правила должны быть **краткими** — AI читает их каждый раз
- Будьте **конкретны** — «Используй camelCase для переменных» лучше, чем «используй хорошие имена»
- **Обновляйте** по мере развития проекта
- **Добавьте в Git** — чтобы вся команда использовала одинаковые правила

## Итоги

- `.cursorrules` определяет, как AI пишет код для проекта
- Разместите в корне проекта
- Включите стиль, фреймворк, тесты и документацию
- Будьте кратки и конкретны

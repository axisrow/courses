# Работа с большими проектами

## Проблема

Большие проекты содержат тысячи файлов. AI не может прочитать всё одновременно. Нужны стратегии для фокусировки.

## Индексация кодовой базы

Cursor **индексирует** проект для быстрого поиска. Это происходит автоматически при открытии папки. Используйте `@codebase` для поиска по индексу.

## Стратегии

### 1. Точные @упоминания
Не полагайтесь на AI в поиске файлов. Указывайте напрямую:
```
@src/auth/middleware.ts @src/auth/types.ts Обнови middleware для поддержки API-ключей
```

### 2. Файл .cursorignore
Создайте `.cursorignore` (аналог `.gitignore`) для исключения файлов:
```
node_modules/
dist/
build/
*.min.js
coverage/
```

Это ускоряет индексацию и делает ответы AI точнее.

### 3. Дробите задачи
Вместо «перепиши всю систему аутентификации»:
1. «Перечисли все файлы, связанные с аутентификацией»
2. «Перепиши функцию входа в auth/login.ts»
3. «Обнови тесты в auth/__tests__/»

### 4. Используйте документацию
```
@docs/architecture.md Помоги добавить новый сервис по этой архитектуре
```

## Советы по производительности

- **Закрывайте неиспользуемые вкладки** — открытые файлы добавляются в контекст AI
- **Конкретные запросы** — `@codebase findUserById` лучше, чем `@codebase user`
- **Обновляйте .cursorignore** — исключайте сгенерированные файлы

## Итоги

- Используйте `@упоминания` для направления AI к конкретным файлам
- Создайте `.cursorignore` для исключения ненужных файлов
- Дробите крупные задачи на мелкие
- `@codebase` для быстрого поиска по проекту

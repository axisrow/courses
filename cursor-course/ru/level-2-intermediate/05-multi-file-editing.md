# Редактирование нескольких файлов

## За пределами одного файла

Реальные проекты содержат десятки и сотни файлов. AI в Cursor может понимать и редактировать **несколько файлов одновременно**.

## Как это работает

Когда вы просите AI внести изменение в чате, он может:
1. **Определить**, какие файлы нужно изменить
2. **Предложить правки** в нескольких файлах
3. **Показать diff** для каждого файла
4. Позволить вам **принять или отклонить** каждое изменение

## Пример: Добавление функции

Вы просите:
```
Добавь функцию «забыл пароль» с формой, API-эндпоинтом и отправкой email
```

Cursor может предложить изменения в:
- `pages/forgot-password.tsx` — Новая страница с формой
- `api/auth/reset.ts` — Новый API-эндпоинт
- `lib/email.ts` — Функция отправки email
- `lib/routes.ts` — Обновлённая конфигурация маршрутов

## Просмотр изменений

1. Вы видите список всех затронутых файлов
2. Нажмите на каждый файл для просмотра diff
3. **Примите** или **отклоните** каждый файл отдельно

## Режим Composer

Для сложных многофайловых задач используйте **Composer** (`Ctrl+I` / `Cmd+I`):
- Создан для многофайловых правок
- Показывает все изменения в одном виде
- Лучше для крупного рефакторинга

## Советы

- Используйте `@упоминания` для указания нужных файлов
- Всегда используйте **Git** — сделайте коммит перед большими изменениями

## Итоги

- Cursor может редактировать несколько файлов за один запрос
- Проверяйте изменения в каждом файле отдельно
- Используйте Composer для сложных многофайловых операций
- Git — ваша страховка

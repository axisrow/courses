# 代理模式 — 完全自主

## 什么是代理模式？

代理模式是 Cursor 最强大的功能。AI 不再等你指挥每一步，而是**主动行动**：
- 阅读和搜索代码库
- 创建和编辑文件
- 运行终端命令
- 安装依赖包
- 运行测试并修复错误

## 如何使用

1. 打开聊天（`Ctrl+L` / `Cmd+L`）
2. 选择 **Agent** 模式（顶部的切换按钮）
3. 用自然语言描述你的任务
4. AI 开始工作——你可以看到每个步骤

## 自主性滑块

如 Andrej Karpathy 所描述，Cursor 有一个"自主性滑块"：
- **Tab 补全** → 最低自主性（你来驱动）
- **Cmd+K** → 中等自主性（你指导，AI 执行）
- **代理模式** → 最高自主性（AI 驱动，你审查）

## 代理模式能做什么

- "搭建一个 Node.js 项目，使用 Express、TypeScript 和 PostgreSQL"
- "为所有 API 端点编写测试"
- "找到并修复导致登录失败的 bug"
- "将认证系统重构为 JWT"

## 安全性：审批操作

代理模式在以下操作前会请求你的**批准**：
- 运行终端命令
- 进行重大更改
- 安装包

最终决定权始终在你手中。

## 最佳实践

1. **从清晰的计划开始** — 预先描述完整任务
2. **审查每个步骤** — 不要盲目接受一切
3. **使用 Git** — 开始前先提交
4. **拆分大任务** — "构建整个应用"太模糊，按功能拆分
5. **提供上下文** — 用 `@` 提及相关文件

## 小结

- 代理模式 = AI 最大自主性
- AI 规划、编码、运行命令、修复错误
- 你审查和批准每个步骤
- 最适合较大的、定义明确的任务

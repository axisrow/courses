# 多文件编辑

## 超越单个文件

真实项目有几十甚至几百个文件。Cursor 的 AI 能理解和**同时编辑多个文件**。

## 工作原理

当你在聊天中请求更改时，AI 可以：
1. **识别**哪些文件需要修改
2. **提出**跨多个文件的编辑
3. **显示** 每个文件的 diff
4. 让你逐个**接受或拒绝**更改

## 示例：添加新功能

你请求：
```
添加"忘记密码"功能，包含邮件表单、API 端点和邮件发送逻辑
```

Cursor 可能会修改：
- `pages/forgot-password.tsx` — 新页面和表单
- `api/auth/reset.ts` — 新 API 端点
- `lib/email.ts` — 邮件发送函数
- `lib/routes.ts` — 更新路由配置

## 审查多文件更改

1. 你会看到所有受影响文件的列表
2. 点击每个文件查看 diff
3. 逐个**接受**或**拒绝**

## Composer 模式

对于复杂的多文件任务，使用 **Composer**（`Ctrl+I` / `Cmd+I`）：
- 专为多文件编辑设计
- 在一个视图中显示所有更改
- 更适合大型重构

## 技巧

- 使用 `@提及` 指向相关文件
- 明确说明你想更改哪些文件
- 仔细审查——AI 可能会修改你没预期的文件
- 使用 **Git**——大型多文件更改前务必提交

## 小结

- Cursor 可以在一个请求中编辑多个文件
- 逐个审查每个文件的更改
- 使用 Composer 处理复杂的多文件操作
- Git 是你的安全网

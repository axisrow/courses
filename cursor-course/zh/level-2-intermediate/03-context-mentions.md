# 使用上下文：@files、@docs、@web

## 为什么上下文很重要

AI 在了解你的工作内容时表现最好。通过 `@` 符号的**上下文提及**，你可以将 AI 指向特定信息。

## 提及类型

### @files — 引用你的代码
```
@utils.py 如何改进这个文件的错误处理？
```

### @folder — 包含整个文件夹
```
@src/components 创建一个匹配现有风格的新组件
```

### @docs — 引用文档
```
@docs 在 Next.js 文档中搜索如何使用服务器组件
```

### @web — 搜索互联网
```
@web 2025年 React 中最新的认证方式是什么？
```

### @codebase — 搜索整个项目
```
@codebase 数据库连接在哪里配置的？
```

## 组合使用

可以在一个提示中使用多个提及：
```
@api/routes.py @models/user.py 按照现有模式添加一个删除用户的端点
```

## 何时使用什么

| 情况 | 使用 |
|------|------|
| 编辑特定文件 | `@filename` |
| 遵循项目模式 | `@folder` |
| 使用框架文档 | `@docs` |
| 需要最新信息 | `@web` |
| 在项目中查找 | `@codebase` |

## 小结

- 使用 `@` 提及给 AI 精确的上下文
- `@files` 用于特定文件，`@web` 用于互联网搜索
- `@codebase` 搜索整个项目
- 更好的上下文 = 更好的 AI 回答

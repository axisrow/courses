# 元提示词

## 介绍

元提示词（Meta-prompting）是 AI 为其他 AI 创建提示词。在 GSD 中，一个轻量级编排器为专门的代理创建任务。

## 操作指南

### GSD 中的元提示词架构

```
你 → 编排器 → 研究员（4个并行）
            → 规划师 → 检查器 → （循环直到通过）
            → 执行者（并行波次）
            → 验证器
```

### 编排器如何工作

1. **从不做重活** — 只负责协调
2. **基于当前状态为子代理创建提示词**
3. **收集结果**并向前传递
4. **保持主窗口上下文** — 你的会话保持在 30-40%

### XML 提示词格式

GSD 使用 XML 来结构化任务，因为 Claude 更好地理解结构化提示词：

```xml
<task type="auto">
  <name>具体名称</name>
  <files>需要修改的文件</files>
  <action>精确指令</action>
  <verify>如何验证</verify>
  <done>完成标准</done>
</task>
```

## 示例

当你运行 `/gsd:plan-phase 1` 时，编排器：
1. 读取 PROJECT.md、REQUIREMENTS.md、CONTEXT.md
2. 为研究员创建提示词："调查技术栈，找到模式"
3. 收集研究 → 为规划师创建提示词
4. 规划师创建 XML 计划
5. 检查器验证计划 → 如果不通过，循环重复

## 总结

- 元提示词 = AI 为 AI 创建提示词
- 编排器协调，不做具体工作
- XML 结构为子代理提供精确指令
- 你的主上下文保持干净

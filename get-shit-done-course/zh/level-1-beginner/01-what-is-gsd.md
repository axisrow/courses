# 什么是 Get Shit Done，为什么需要它

## 介绍

你在 Claude Code 中写了一个提示词，得到了很好的结果。又写了一个。再写一个。但在 30-40 条消息之后，质量开始下降——回复变短，代码变差，模型"忘记"了早期的决策。这叫做 **context rot**（上下文退化）。

**Get Shit Done (GSD)** 是一个元提示词和上下文工程系统，解决了这个问题。它在正确的时间给 Claude（或其他 AI 代理）提供正确的上下文。

## GSD 的功能

- 将工作拆分为原子任务，每个任务使用全新的上下文窗口
- 通过 markdown 文件管理项目状态
- 编排子代理：研究员、规划师、执行者
- 自动提交每个任务

## Context Rot 问题

| 没有 GSD | 使用 GSD |
|---------|---------|
| 一个长对话 | 多个短会话 |
| 上下文填满 | 每个任务获得全新的 200k tokens |
| 质量下降 | 质量稳定 |
| 手动管理 | 自动编排 |

## 适用人群

GSD 为那些想描述想法并正确构建的人而设计——不需要假装在管理一个 50 人的工程团队。

## 总结

- GSD 是用于 Claude Code、OpenCode 和 Gemini CLI 的元提示词系统
- 解决上下文退化（context rot）问题
- 将工作拆分为具有干净上下文的原子任务
- 不需要编程知识

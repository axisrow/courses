# Shannon架构

## 简介

Shannon不是一个单一的"机器人"——它是一个由专门的AI智能体组成的系统，通过四个不同的阶段协同工作。

## 架构概览

```
                ┌──────────────────────┐
                │       侦察           │
                └──────────┬───────────┘
                           │
            ┌──────────────┼──────────────┐
            ▼              ▼              ▼
    ┌──────────────┐ ┌──────────────┐ ┌──────────┐
    │ 分析（注入）  │ │ 分析（XSS）  │ │   ...    │
    └──────┬───────┘ └──────┬───────┘ └────┬─────┘
           ▼                ▼              ▼
    ┌──────────────┐ ┌──────────────┐ ┌──────────┐
    │ 利用（注入）  │ │ 利用（XSS）  │ │   ...    │
    └──────┬───────┘ └──────┬───────┘ └────┬─────┘
           └────────┬───────┴──────────────┘
                    ▼
            ┌──────────────────────┐
            │        报告           │
            └──────────────────────┘
```

## 第1阶段：侦察
构建全面的地图：源代码分析、Nmap/Subfinder/WhatWeb扫描、浏览器探索。产出详细的攻击面地图。

## 第2阶段：漏洞分析
专门的智能体**并行**搜索漏洞——每个OWASP类别一个。它们执行数据流分析，追踪用户输入到危险汇点。产出**可利用路径假设**列表。

## 第3阶段：利用
专门的智能体使用浏览器自动化、CLI工具和自定义脚本尝试真实攻击。**"无利用，无报告"**——失败的假设被丢弃。

## 第4阶段：报告
将所有验证的发现编译成专业报告，附带可重现的PoC。

## 技术栈

- **Anthropic Claude Agent SDK** — AI推理核心
- **Temporal** — 工作流编排
- **Docker** — 隔离和可重现性
- **浏览器自动化** — 真实浏览器交互
- **Nmap、Subfinder、WhatWeb、Schemathesis** — 侦察工具

## 总结

- Shannon是一个具有4个阶段的多智能体系统
- 智能体并行工作以提高速度
- 严格的"无利用，无报告"策略消除误报

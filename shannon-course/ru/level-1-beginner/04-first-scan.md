# Первый скан

## Введение

Давайте проведём полноценный пентест на учебном приложении OWASP Juice Shop — специально созданном «дырявом» приложении.

## Шаг 1: Запуск Juice Shop

```bash
docker run -d -p 3000:3000 bkimminich/juice-shop
```

Откройте `http://localhost:3000` — вы увидите интернет-магазин с соками.

## Шаг 2: Подготовка исходного кода

```bash
git clone https://github.com/juice-shop/juice-shop.git ./repos/juice-shop
```

## Шаг 3: Запуск Shannon

```bash
./shannon start URL=http://host.docker.internal:3000 REPO=juice-shop
```

Shannon вернёт ID workflow, например: `shannon-1771007534808`.

## Шаг 4: Наблюдение за процессом

```bash
# Логи в реальном времени
./shannon logs

# Или откройте веб-интерфейс
open http://localhost:8233
```

Процесс занимает примерно 1–1.5 часа. Shannon пройдёт через 4 фазы:
1. **Разведка** — изучит приложение, найдёт точки входа
2. **Анализ уязвимостей** — найдёт потенциальные проблемы в коде
3. **Эксплуатация** — попытается реально использовать каждую уязвимость
4. **Отчёт** — соберёт доказанные находки в отчёт

## Шаг 5: Получение результатов

После завершения отчёт появится в папке:

```
audit-logs/{hostname}_{sessionId}/deliverables/
```

## Использование воркспейсов

Если тест прервался, можно продолжить с того же места:

```bash
# Именованный воркспейс
./shannon start URL=http://host.docker.internal:3000 REPO=juice-shop WORKSPACE=my-test

# Возобновление
./shannon start URL=http://host.docker.internal:3000 REPO=juice-shop WORKSPACE=my-test
```

Shannon определит, какие агенты уже завершили работу, и продолжит с того места, где остановился.

## Итоги

- Вы запустили полноценный пентест на учебном приложении
- Увидели 4 фазы работы Shannon
- Научились использовать воркспейсы для возобновления тестов
- Результаты сохраняются в папке `audit-logs/`

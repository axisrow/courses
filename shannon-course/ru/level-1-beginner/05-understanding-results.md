# Понимание результатов

## Введение

Shannon завершил тест — теперь разберёмся, как читать отчёт и что с ним делать.

## Структура выходных данных

```
audit-logs/{hostname}_{sessionId}/
├── session.json          # Метрики и данные сессии
├── agents/               # Логи каждого агента
├── prompts/              # Снапшоты промптов
└── deliverables/
    └── comprehensive_security_assessment_report.md   # Главный отчёт
```

Самый важный файл — `comprehensive_security_assessment_report.md`. Это финальный отчёт.

## Как читать отчёт

Отчёт содержит только **доказанные уязвимости**. Каждая находка включает:

### 1. Описание уязвимости
Что именно найдено и какой это тип (SQL Injection, XSS, SSRF и т.д.)

### 2. Уровень критичности
- **Critical** — можно украсть все данные или получить полный контроль
- **High** — серьёзная проблема, нужно чинить срочно
- **Medium** — проблема есть, но эксплуатация сложнее
- **Low** — незначительный риск

### 3. Proof of Concept (PoC)
Самая ценная часть — готовый код или команда для воспроизведения атаки. Вы можете скопировать и выполнить PoC, чтобы убедиться самостоятельно.

### 4. Рекомендации по исправлению
Что нужно изменить в коде, чтобы закрыть уязвимость.

## Пример находки

```markdown
## SQL Injection в /api/products/search

**Критичность:** Critical

**Описание:** Параметр `q` в эндпоинте поиска не фильтруется
и позволяет выполнить произвольный SQL-запрос.

**PoC:**
curl "http://target.com/api/products/search?q=test')) UNION SELECT
  id,email,password,role,... FROM Users--"

**Результат:** Получена полная таблица пользователей
с хешами паролей.

**Рекомендация:** Используйте параметризованные запросы
вместо конкатенации строк.
```

## Что делать с результатами

1. **Расставьте приоритеты** — исправляйте Critical и High в первую очередь
2. **Воспроизведите** — используйте PoC чтобы проверить уязвимость
3. **Исправьте** — внесите изменения в код
4. **Перепроверьте** — запустите Shannon снова, чтобы убедиться, что уязвимость закрыта

## Логи агентов

В папке `agents/` находятся подробные логи работы каждого агента. Это полезно для:
- Понимания, как Shannon нашёл уязвимость
- Отладки, если что-то пошло не так
- Изучения методологии тестирования

## Итоги

- Главный файл — отчёт в папке `deliverables/`
- Каждая находка включает описание, критичность, PoC и рекомендации
- Все уязвимости в отчёте — доказанные (реально эксплуатированные)
- Исправляйте Critical и High в первую очередь, затем перезапускайте тест

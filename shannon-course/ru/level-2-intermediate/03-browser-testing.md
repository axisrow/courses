# Browser-based тестирование

## Введение

Shannon не просто анализирует код — он открывает настоящий браузер и взаимодействует с приложением как человек. Это позволяет находить уязвимости, которые видны только при реальном использовании.

## Как это работает

Shannon использует автоматизацию браузера для:

1. **Входа в приложение** — заполняет формы, нажимает кнопки, проходит 2FA
2. **Навигации** — переходит по страницам, находит скрытые эндпоинты
3. **Эксплуатации** — выполняет XSS-атаки, проверяет Cookie, тестирует перенаправления
4. **Доказательства** — делает скриншоты и сохраняет ответы как доказательство

## Что тестируется через браузер

### XSS-атаки
Браузер нужен, чтобы подтвердить, что внедрённый JavaScript действительно выполняется. Shannon вставляет скрипт и проверяет результат в реальном браузере.

### Аутентификация
Shannon проходит весь процесс входа: заполнение форм, обработка TOTP, OAuth-потоки (Sign in with Google).

### Клиентские уязвимости
- DOM-based XSS
- Clickjacking
- Небезопасное хранение данных в браузере
- Перенаправление URL

## Сочетание с анализом кода

Shannon объединяет два подхода:

| Подход | Что даёт |
|--------|----------|
| **White-box** (анализ кода) | Находит потенциальные уязвимости в исходном коде |
| **Black-box** (браузер) | Подтверждает, что уязвимость реально эксплуатируема |

Это сочетание называется **code-aware dynamic testing** — анализ кода направляет атаки, а браузер их доказывает.

## Встроенные инструменты

Помимо браузера, Shannon использует:
- **Nmap** — сканирование портов и сервисов
- **Subfinder** — поиск поддоменов
- **WhatWeb** — определение технологий
- **Schemathesis** — тестирование API по схеме

## Итоги

- Shannon использует реальный браузер для эксплуатации уязвимостей
- Браузер подтверждает XSS, проходит аутентификацию, проверяет клиентские баги
- Сочетание анализа кода и браузерного тестирования даёт максимальную точность

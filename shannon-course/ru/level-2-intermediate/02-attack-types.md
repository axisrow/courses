# Типы атак

## Введение

Shannon проверяет несколько категорий уязвимостей. Разберём каждую простым языком.

## SQL Injection (Инъекция SQL)

**Что это:** Злоумышленник вставляет SQL-код в поля ввода (поиск, логин), и сервер выполняет его как часть запроса к базе данных.

**Пример:** В поле поиска вводят `' OR 1=1 --`, и сервер отдаёт все записи из таблицы.

**Чем опасно:** Кража всех данных, удаление базы, получение паролей.

**Shannon находит:** SQL-инъекции и слепые SQL-инъекции (Blind SQLi), где результат не виден напрямую.

## Cross-Site Scripting (XSS)

**Что это:** Злоумышленник внедряет JavaScript-код на страницу, который выполняется в браузере других пользователей.

**Типы:**
- **Reflected** — вредоносный код в URL, срабатывает при переходе по ссылке
- **Stored** — код сохраняется на сервере (например, в комментарии) и срабатывает у всех посетителей
- **DOM-based** — код манипулирует DOM-деревом страницы

**Чем опасно:** Кража cookie, перенаправление на фишинг, действия от имени жертвы.

## Server-Side Request Forgery (SSRF)

**Что это:** Злоумышленник заставляет сервер отправить запрос на внутренний ресурс, к которому он не должен иметь доступа.

**Пример:** Функция «загрузить картинку по URL» используется для обращения к `http://localhost:8080/admin`.

**Чем опасно:** Доступ к внутренним сервисам, кража токенов, разведка внутренней сети.

## Broken Authentication & Authorization

**Что это:** Ошибки в системе аутентификации (кто вы) и авторизации (что вам можно).

**Примеры:**
- Обход логина
- Эскалация привилегий (обычный пользователь → администратор)
- IDOR (доступ к чужим данным, подменяя ID)
- Атаки на JWT-токены

**Чем опасно:** Полный контроль над аккаунтами других пользователей.

## Command Injection (Инъекция команд)

**Что это:** Злоумышленник внедряет команды операционной системы через поля ввода.

**Пример:** Поле «ping хоста» принимает `google.com; cat /etc/passwd`.

**Чем опасно:** Полный контроль над сервером.

## Что Shannon не покрывает (пока)

- Уязвимости в сторонних библиотеках
- Слабое шифрование
- Небезопасные конфигурации (если их нельзя эксплуатировать)
- Бизнес-логика

Эти типы анализа доступны в Shannon Pro.

## Итоги

- Shannon покрывает 5 основных категорий атак: SQLi, XSS, SSRF, Broken Auth, Command Injection
- Каждая уязвимость доказывается реальным эксплойтом
- Для расширенного покрытия существует Shannon Pro
